## 2 · 关键工程实现(已落地)

### 2.1 git 单向 fetch 镜像(方案 C 第 1 步)

```bash
cd ~/openclaw/workspace/V2.1
git remote -v
# main_repo  ~/openclaw/workspace/yiyu-thinktank-workbench (fetch)
# main_repo  no_push_v21_lab_red_line (push)    ← 禁 push, 红线保护
```

**同步主仓库 bug fix**(主仓库每个 commit 后跑一次):

```bash
cd ~/openclaw/workspace/V2.1
git fetch main_repo
git log --oneline v2.2-arch-v2..main_repo/main   # 看主仓库新增 commits
git cherry-pick <commit_hash>                     # 逐个 cherry-pick
# 或一次性 cherry-pick 多个
git cherry-pick <hash1> <hash2> <hash3>
```

**冲突处理**:
- 多数 bug fix 是改 view 文件,不会跟 V2.1 lab (改 collector / generator) 冲突
- 真冲突时:V2.1 已有同等功能 → `git cherry-pick --skip`;V2.1 需要更新 → 手动 merge

### 2.2 ENV-gated 双 app 隔离(方案 C 第 2-3 步)

V2.1 lab 用 `YIYU_LAB_MODE=1` 环境变量触发:

| 配置 | 主仓库 (默认) | V2.1 lab (`YIYU_LAB_MODE=1`) |
|---|---|---|
| Backend port | 47829 | 47831 |
| Cloud backend port | 47830 | 47832 |
| Electron productName | 益语智库自用平台 V2.0 | 益语智库 V2.1 Lab |
| Bundle ID | com.yiyu.selfworkbench2 | com.yiyu.selfworkbench2.v21lab |
| userData dir | YiyuThinkTankWorkbench2 | YiyuThinkTankWorkbench2_V21Lab |

实现位置:
- `src/main/main.ts:25-45` — Electron 启动配置
- `src/main/preload.ts:13-16` — frontend → backend URL

**关键性质**: 默认行为(无 ENV)完全跟主仓库一致 → 主仓库 cherry-pick 我们的代码也不破坏主仓库行为。

## 5 · 红线 + 风险防护

### 5.1 V2.1 红线(永不破)

1. ❌ **绝不 push V2.1 commits 到主仓库** — git remote push URL 已设 `no_push_v21_lab_red_line`,强制 push 会失败
2. ❌ **绝不在主仓库直接改 V2.1 实验性代码** — 主仓库代码改动只能是 v2.0 bug fix + v2.2 工具层(IngestPipeline / IdempotencyStore / ReasoningTraceStore)
3. ❌ **不在主仓库加 LAB_MODE 分支逻辑** — LAB_MODE 是 V2.1 lab 独有的隔离机制,主仓库不需要

### 5.2 主仓库变更同步流程

1. Cowork Claude / 顾源源 在主仓库 commit
2. 我(A AI) 在 V2.1 跑 `git fetch main_repo`
3. 看 `git log --oneline v2.2-arch-v2..main_repo/main` 决定哪些 cherry-pick
4. `git cherry-pick <hash>` 逐个 pick
5. 冲突时:V2.1 优先(因为 V2.1 是实验更靠前)

### 5.3 风险

| 风险 | 兜底 |
|---|---|
| V2.1 cherry-pick 冲突积累 | 每天同步一次, 不要积压 |
| V2.1 LLM 调用占用主仓库 LLM 配额 | 用相同 ai_service, 共用配额, 跑前评估 |
| 双 app 同跑内存翻倍 | M3 Max 64GB 够, 但避免同时跑大批量 LLM 任务 |
| db 路径写错把主仓库 db 覆盖 | init_v21_db_from_main.sh 只 cp 不 mv, 主仓库 db 0 风险 |
