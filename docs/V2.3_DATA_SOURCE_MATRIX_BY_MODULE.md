## 0 · 7 维度 定义(顾源源原话精确化)

| 维度 | 定义 | 取值 |
|---|---|---|
| **D1 数据类型** | 这条信息是用户主动 / AI 抽 / 系统记录 / 外部抓取 | user_active / ai_extracted / system_log / external_crawl |
| **D2 数据程度** | 颗粒度 + 完整度 | atomic / structured / freetext / referenced |
| **D3 数据周期** | 多久产生一次 + 多久过期 | realtime / daily / weekly / monthly / permanent |
| **D4 置信度** | 0-1 真实度 + 该信多少 | high(≥0.85) / medium(0.6-0.85) / low(<0.6) |
| **D5 数据性质** | content_role | fact / decision / risk / observation / plan / lesson / quote / commitment / speculation |
| **D6 关键词类型** | 这条数据通常含什么类别的关键词 | person / org / project / time / money / action / status / risk_signal / location |
| **D7 当前去向** | 这数据现在写到哪表 + 是否走 IngestPipeline | table_name + path (yes/no) |

## 11 · ★ 交叉印证机制设计(顾源源核心问题答案)

### 11.1 跨板块同类关键词印证(应该撞却没撞的)

#### 例 1 · "兴盛" 项目跨板块印证(实测)

```
板块 02 工作台 → smart_file_import 抽 atomic_facts: subject="兴盛" 多条
板块 03 战略陪伴 → narrative_generator 6 段提及"兴盛"
板块 05 成长中心 → handbook_entries 是否含"兴盛"? (查实)
板块 08 计划工坊 → project_modules 是否有 name="兴盛"? (查实)

→ 当前: 4 板块 N 个表都可能含"兴盛", 没机制确认是"同一项目"
→ V2.3 加: cross_reference_check.py — 跨表搜同关键词 → 关联
```

#### 例 2 · 同音字 "心灵" vs "心理" (B AI K-3 经典案例)

```
板块 02 (smart_file_import 抽): subject="心灵魔法学院"
板块 04 (intelligence 抓取): subject="心理魔法学院"
当前 atomic_facts.detect_update_relation 用 SQL LIKE → 永不撞
应加: pinyin 检测 → 嫌疑度 0.95 → 进 clarification_records → 用户裁决
```

#### 例 3 · 人物跨板块 "张真"

```
板块 01 任务 (1.1 创建): "和张真进行 5月对齐会"
板块 02 工作台 (2.1 对话): "张真昨天说..."
板块 03 战略陪伴 (3.5 思考): "张真接任理事长"
板块 06 智能导入 (6.1 讲述): "张真本人偏感性"

→ 当前: 4 板块各自存, 不知是同一人
→ V2.3 加: entities (4987 条) 跨板块 person 实体唯一化, 任何板块写"张真" 都关联同 entity_id
```

### 11.2 印证机制 4 层(从最具体到最抽象)

```
Layer 1 · 字面相同 (string_eq)
  · 已实现 (detect_update_relation 现状)
  · 触发率 0.1% (实测)

Layer 2 · 同音字 (pinyin similarity)
  · 待实现 (V2.3 P1.B)
  · 期望解 "心灵 vs 心理" 类问题
  · 实现: pypinyin + Levenshtein

Layer 3 · 语义相似 (embedding similarity)
  · 待实现 (V2.3)

```

### 11.3 数据完善机制(印证后怎么做)

| 印证结果 | 系统动作 |
|---|---|
| Layer 1 字面相同 + 同 source | 不写新事实(去重) |
| Layer 1 相同 + 跨 source | update_relation='complement' 写新事实 |
| Layer 2 同音字 高嫌疑 | 写 clarification_records + 弹用户裁决 |
| Layer 3 语义相似 ≥ 0.85 | 写 entity_merge_log 候选 + 待 L3 自校验确认 |
| Layer 4 LLM 判同 | 自动 alias 加入 client_glossary |
