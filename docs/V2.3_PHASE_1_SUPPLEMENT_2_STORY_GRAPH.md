期望: 22 条数据,~5 种 predicate(主要 mention/located_at 等基础关系)。

**蓝图 § 四 机制四 12 类关系**:

| 关系类型 | 含义 | 现状 predicate? |
|---|---|---|
| 参与 (participate_in) | 人物参与项目/事件 | ⚠️ 可能用 mentions |
| 负责 (owns / responsible_for) | 谁负责任务/项目 | ⚠️ tasks.owner_id 隐式 |
| 决策 (decides) | 谁做了某个决定 | ❌ 无 |
| 承诺 (commits_to) | 谁承诺什么 | ⚠️ commitments 表隐式 |
| 影响 (impacts) | A 影响 B | ❌ 无 |
| 支持 (supports / corroborates) | 一份证据支持一个事实 | ❌ 无 |
| 反驳 (contradicts) | 一份证据反驳另一份 | ⚠️ fact_contradictions 隐式 |
| 更新 (supersedes) | 新版替代旧版 | 🟢 atomic_facts.superseded_by_id 有 |
| 依赖 (depends_on) | 任务依赖任务 / 决策依赖事实 | ❌ 无 |
| 派生 (derived_from) | A 派生于 B | ⚠️ atomic_facts.derived_from_ids_json |
| 关联 (associates_with) | 通用弱关联 | 🟢 entity_mentions 12184 |

**真信号**:
- 12 类关系当前**只 5-6 类有实现**(更新/关联/派生 部分 / 反驳 隐式)
- **决策 / 影响 / 支持 / 依赖 / 触发行动 这 5 类是 V2.3 核心 gap**
- 尤其"触发行动"是蓝图 § 六线索四"行动结果线"的真灵魂

**实施方式**: 不复制数据到 entities 表,而是 **entity_link 视图**(SQL VIEW)统一访问:

```sql
-- 概念视图 (V2.3 阶段 4 实施):
CREATE VIEW story_node AS
  SELECT id, entity_type, normalized_name AS label, client_id, created_at, 'entities' AS table_source
  FROM entities
  UNION ALL
  SELECT id, 'project' AS entity_type, name AS label, client_id, created_at, 'project_modules'
  FROM project_modules
  UNION ALL
  SELECT id, 'task' AS entity_type, title AS label, client_id, created_at, 'tasks'
  FROM tasks

  ;

## 4 · 扩展设计 · relationship_triples.predicate 枚举(V2.3)

**新加 12 类 predicate**:
```
participate_in    人物参与项目/事件
responsible_for   人物对任务/项目负责
decides          人物对某事做决定
commits_to       人物对客户承诺
impacts          A 影响 B (战略/业务影响)

## 5 · 故事图谱构建 · 自动从业务表抽

**V2.3 阶段 4 实施**: 写 `backend/app/services/story_graph_builder.py`:

```python
def build_story_graph_for_client(db, client_id):
    """从业务表抽节点 + 关系, 生成项目故事图谱."""

    # 1. 抽节点 (12 类)
    nodes = []
    nodes.extend(_load_persons(db, client_id))         # entities WHERE type='person'
    nodes.extend(_load_projects(db, client_id))        # project_modules + event_lines
    nodes.extend(_load_documents(db, client_id))       # v2_documents
    nodes.extend(_load_tasks(db, client_id))           # tasks
    nodes.extend(_load_commitments(db, client_id))    # commitments
    nodes.extend(_load_decisions(db, client_id))      # decisions + key_decisions
    nodes.extend(_load_risks(db, client_id))           # risk_signals
    nodes.extend(_load_judgments(db, client_id))      # strategic_thought_insights
    nodes.extend(_load_output_artifacts(db, client_id))
    nodes.extend(_load_external_intel(db, client_id))
    nodes.extend(_load_method_cards(db, client_id))

    # 2. 抽关系 (12 类)
    edges = []
    # 显式关系 (relationship_triples)
    edges.extend(_load_explicit_relations(db, client_id))
    # 隐式关系 (从业务表外键推):
    edges.extend(_load_task_owns_edges(db, client_id))         # tasks.owner_id → person (responsible_for)
    edges.extend(_load_commitment_committer_edges(db, client_id))  # commitments.committer → person (commits_to)
    edges.extend(_load_task_plan_edges(db, client_id))         # task_plan_links → project (participate_in)
    edges.extend(_load_fact_supersedes_edges(db, client_id))   # atomic_facts.superseded_by_id (supersedes)
    edges.extend(_load_fact_contradiction_edges(db, client_id))  # fact_contradictions (contradicts)
    # ★ V2.3 新加 (triggers_action):
    edges.extend(_load_intel_to_task_edges(db, client_id))     # intelligence → task (转任务)
    edges.extend(_load_risk_to_meeting_edges(db, client_id))   # risk → meeting
    edges.extend(_load_clarification_to_fact_update_edges(db, client_id))

    return StoryGraph(client_id=client_id, nodes=nodes, edges=edges)
```

**蓝图 § 六 线索四 行动结果线**: 重点抓 triggers_action 边的真触发率。
