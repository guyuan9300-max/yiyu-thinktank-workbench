# 三层入口与场景提醒设计稿

> 对齐说明（2026-03-21）：
> 本文档继续有效，但当前阶段仅作为 `P2` 的任务与日历浅入口设计储备。
> `深入口` 归组织启动中心，`浅入口 + 提醒入口` 归任务与日历，
> 实施顺序统一服从 [AI 背景优先总规划](./ai-context-first-rollout-plan.md)
> 与 [全局模块改造清单与阶段路线图](./global-module-refactor-roadmap.md)。

## 目标

解决一个核心矛盾：

- 战略、DNA、季度计划这类内容很重要，但频率低，应该放深
- 月度计划、流程沉淀这类内容频率高，必须放浅
- 任务与日历不能因为承接这些能力而变得很重

所以需要把入口拆成三层：

1. 深入口
2. 浅入口
3. 提醒入口

## 一、三层入口模型

## 1. 深入口

定义：

- 低频
- 结构性强
- 全局真源头
- 通常由 CEO 或组织管理员维护

承载内容：

- 公司 DNA
- 公司介绍 / 业务介绍 / 组织介绍 / 市场调研
- 年度战略
- 季度机构计划
- 部门季度计划框架

推荐放置：

- `组织搭建中心`
- 更准确地说：`组织启动中心` 里的“战略与计划”

## 2. 浅入口

定义：

- 高频
- 工作中会经常进入
- 需要尽量少层级、少跳转

承载内容：

- 部门月度计划
- 周计划承接
- 流程沉淀
- 模板补充
- 方法补充

推荐放置：

- `任务与日历`
- 具体任务节点
- 会议结束节点
- 周总结节点

## 3. 提醒入口

定义：

- 不是主编辑页
- 用来发现前置缺失
- 用来把用户拉到正确入口

承载内容：

- 缺月度计划提醒
- 缺季度计划提醒
- 缺流程沉淀提醒
- 月度计划未拆任务提醒

推荐放置：

- `任务与日历` 首页
- 周计划区
- 周总结区
- 会议结束页
- 任务完成后的轻提示

## 二、按角色划分入口

## CEO / 组织管理员

入口重点：

- 深入口

主要动作：

- 搭组织架构
- 指定负责人
- 发邀请码 / 未来发二维码
- 上传公司 DNA
- 上传年度战略
- 审核 AI 拆出的季度计划

不要求他在任务页高频维护月度计划。

## 部门负责人

入口重点：

- 深入口 + 浅入口

主要动作：

- 在组织启动中心承接部门季度计划
- 在任务与日历里维护月度计划
- 在周计划里承接任务

也就是说，部门负责人是两个入口都会用的人。

## 普通员工

入口重点：

- 浅入口 + 提醒入口

主要动作：

- 上传流程
- 上传模板
- 沉淀经验
- 从任务和会议节点补资产

普通员工不应该频繁碰战略层。

## 三、任务与日历里的具体承载方式

任务与日历不能加一个很重的新模块，否则会破坏现在的使用体验。

所以建议只加三个轻层承载点。

## 1. 顶部计划状态条

放置位置：

- 任务主看板顶部
- 日历主视图顶部

作用：

- 告诉用户当前月计划状态
- 告诉用户当前是否已经承接到周
- 告诉用户是否已拆成任务

显示文案建议：

- `本月计划未建立`
- `本月计划待补`
- `本月计划已建立，待拆任务`
- `本月计划已承接到本周`

按钮建议：

- `补本月计划`
- `查看本月重点`
- `从本月计划生成任务`

## 2. 周总结前置提醒区

不是把月度计划放进总结正文里，而是在总结区前面放一个前置提醒层。

目的：

- 在用户进入总结之前，先告诉他哪些上游缺失会影响总结质量

提醒示例：

- “当前部门还没有本月计划，复盘会缺主线。”
- “本周任务未挂接月度重点，建议先补挂接。”
- “本周出现 3 条重复流程，建议补流程沉淀。”

按钮：

- `去补本月计划`
- `挂接本周任务`
- `沉淀为流程`

## 3. 事件型轻入口

在这些节点露出补录入口：

- 任务完成后
- 会议结束后
- 周总结后
- 复盘完成后

动作示例：

- `补本月重点`
- `沉淀为流程`
- `记录为模板`
- `关联到本季度目标`

## 四、提醒入口的触发规则

提醒不应该泛滥，要按缺失场景来触发。

## 1. 缺季度计划

触发条件：

- 用户进入任务与日历
- 当前部门没有季度计划

提示：

- “当前部门还没有季度计划，请先去组织启动中心补季度目标。”

按钮：

- `去补部门季度计划`

## 2. 缺月度计划

触发条件：

- 已有部门季度计划
- 当前月还没有月度计划

提示：

- “这个月还没有部门计划，任务建议会偏泛。”

按钮：

- `去补本月计划`
- `稍后`

## 3. 月度计划未拆任务

触发条件：

- 已有本月计划
- 但当前月没有挂接任务

提示：

- “本月重点已经有了，但还没拆成任务。”

按钮：

- `从计划生成任务`

## 4. 缺流程沉淀

触发条件：

- 一周内存在重复执行动作
- 但没有流程或模板资产

提示：

- “本周已有重复动作，建议沉淀为流程模板。”

按钮：

- `去补流程`

## 五、不要怎么做

1. 不要把月度计划藏在总结底部
2. 不要让用户必须先进系统设置才能维护高频内容
3. 不要在任务页加一个大而全的新二级系统
4. 不要让所有角色看到完全相同的入口结构
5. 不要把“提醒入口”做成频繁弹窗

## 六、P1 / P2 对应关系

## P1

只落：

- 深入口
- 组织战略与季度计划结构
- 部门季度计划结构
- 任务页未来提醒所需的缺口判断前置

P1 不落：

- 任务页浅入口
- 月度计划编辑
- 流程沉淀浅入口

## P2

再落：

- 顶部计划状态条
- 周总结前置提醒区
- 任务/会议节点的事件型轻入口
- 部门月度计划编辑和任务承接

## 七、最小闭环

P2 第一版只建议先实现这 4 个浅入口：

1. `去补部门季度计划`
2. `去补本月计划`
3. `从本月计划生成任务`
4. `沉淀为流程`

只要这 4 个入口成立，任务页就会明显更聪明，但不会变重。
