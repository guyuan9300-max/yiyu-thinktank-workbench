## 已知 caveat

1. **dim_11 性能数字基于本地 SQLite + 单一查询模式**:生产数据量(11 个客户)级别下性能极佳,但 v2.1 数据量上升后(W2 加更多模块)需要重测
2. **dim_8 真实 e2e 依赖外部火山云可用**:如果云端宕机,这个维度会 FAIL(不是误报,是真实故障)
3. **并发 sync 未覆盖**:`sync_organization_directory` 用了 `db._lock`,但没专门测试两次并行 sync 的行为 — Week 2 可补
4. **测试 mark 警告**:11 个 `Unknown pytest.mark.integration` warning(只是 mark 未注册,不影响运行),可在 `pytest.ini` 里 register 掉
