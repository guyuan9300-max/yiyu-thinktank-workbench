## 0 · 计划的设计原则(2 条新原则)

### 原则 1 · 按"可感知度"排序(顾源源北极星驱动)

不按工作量排,按"几小时后顾源源能在 app 里看到什么改变"排:

```
最大杠杆点 (10 行改动 → 整链联动) 最优先
→ 北极星 1-2 个里程碑能看到结果 的次之
→ 工作量大但效果远期的最后
```

### 原则 2 · 跟 B 全量盘点 systematic 交叉印证

B 5/22 提案 P3-P6 按工作量排(smart_file_import → 路径 2 → ...),A 改按可感知度排:

```

P3 smart_file_import (1天) →     M-A broadcast 接通 (30 min) ← 缺位补充
P4 路径 2 接入 (2-3天) →         M-B 路径 2 接入 (2-3 h) ← 直接打 5/19 命中
```

A 排序的优势: 同样 5-7 小时,5/19 命中 1/7 → 5/7+ **首日可见**(B 排序要等 P4 完才看到)。

## 1 · 5 个里程碑(M-A 到 M-E 一天内完成 + M-F 留 V2.3)

### M-A · broadcast 接通 IngestPipeline(★ 最大杠杆,30 min)

**Why**: data_center_broadcast.py 已实现 L1+L2 异步钩子(778 行 shadow run 数据证实在工作),但 IngestPipeline.ingest() 写完后不调它。**一个 10 行改动让 4 路径全部触发整链**。

**改动**(主仓库 `backend/app/services/ingest_pipeline.py`):

```python
# 在 ingest() 末尾(写完 atomic_facts + event_log + ai_episode_log 之后)
from app.services.data_center_broadcast import broadcast_data_changed

if result.written:
    broadcast_data_changed(
        client_id=req.client_id,
        reason=f"ingest_pipeline:{req.path}",
        fact_ids=[result.fact_id],
    )
```

**验收**:
- IngestPipeline 写一条 atomic_fact → broadcast 触发 L1 同步入队 analysis_job
- 60-90 秒后 narrative_stale_signals 标记
- 战略陪伴页面 useEffect 检测到 stale → 自动重新拉 6 段叙事

**这一步通了之后,后续 M-B/C/D 每个路径接通,顾源源 60-90 秒内就能在 app 里看到刷新效果**。

### M-B · 路径 2 接入 IngestPipeline(2-3 h)

**Why**: 5/19 张真会议 7 道金标准 **全是路径 2 数据**(法人/秘书长/兴盛+心理魔法学院都在 meetings + tasks + commitments)。路径 2 接通 = 5/19 命中 1/7 → 5/7+,直接打北极星。

**改动**(主仓库):

```
1. backend/app/modules/task/ 写任务时调 IngestPipeline.ingest(path='task_review', sub_kind='task')
2. backend/app/modules/commitment/ 写承诺时同上
3. 加 backend/app/services/event_line_activities_to_ingest.py (新增 service)
   把 event_line_activities 表的 task_activity / document_ingest / attachment
   过一遍 IngestPipeline (一次性 backfill + 后续增量)
4. weekly_reviews + meetings + decisions + action_items 同步走 IngestPipeline
```

**验收**:
- atomic_facts 新增 ≥ 300 条(路径 2 backfill)
- atomic_facts update_relation 不再 100% 'none'(出现 supersedes / complement)
- 跑 dogfood: 5/19 命中 ≥ 5/7

### M-C · narrative_collector 补 atomic_facts + 8 张漏拉表(1-2 h)

**Why**: A 5/22 dogfood 实证 collector 拉 0 条 atomic_facts。M-B 接通后 atomic_facts 有 ~600 条路径 2 数据,collector 必须拉到。

**改动**(主仓库 `backend/app/services/narrative_collector.py`):

```python
def collect_client_fact_bundle(db, client_id, ...) -> ClientFactBundle:
    # 新增:
    bundle.atomic_facts = _collect_atomic_facts(db, client_id, limit=50)      # ★ 当前 0 → 50
    bundle.event_line_activities = _collect_main_timeline(db, client_id)       # 加 activity_logs 26677 主时间线

    bundle.client_dna = _collect_client_dna(db, client_id)                     # client_dna_documents
    bundle.strategic_profile = _collect_strategic_profile(db, client_id)
```

**验收**:
