# 官网分发与应用内更新总计划

## 目标

- 首发模式：官网分发版，不走 Mac App Store。
- 安装体验：用户第一次从官网下载安装包，双击后可以安装并正常运行。
- 更新体验：用户安装后，软件内可以提示新版本、下载新版，或在必要时引导安装。
- 发版纪律：发布动作由受控构建机或 CI 执行，不在普通用户机器上完成签名、公证和上传。

## 当前结论

- 当前仓库已经能产出 `DMG`，但还不是正式可分发版。
- 当前桌面端启动仍依赖：
  - 本机存在 `uv`
  - 首次启动创建 `backend` 与 `cloud_backend` 的运行时 venv
  - 启动时执行 `uv sync`
- 这说明当前安装包更像“开发者可运行包”，还不是“普通用户无感安装包”。

| P5 | 做回滚与灰度 | 稳定版/测试版分渠道；保留旧包 | 回滚操作文档、版本保留策略、灰度发布规则 | 软件里可显示“当前渠道”和“切换渠道提示” | 发布出问题时，可把更新源切回上一版本 |

## 当前实施顺序

1. 先在仓库内落计划文档、资源目录说明和软件内更新入口。
2. 再收口打包链路，保证 `dist:mac` 稳定成功。
3. 再补签名、公证和官网更新源。
4. 最后才接真正的应用内自动更新。
