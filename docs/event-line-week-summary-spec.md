# 事件线式周判断规格

## 1. 目标

这份规格只解决一个问题：

`如何把周总结从任务流水账，升级成面向管理者可读、面向 AI 可推理的事件线式周判断。`

它服务两类核心对象：

- 部门负责人：快速建立团队本周发生了什么、哪里需要干预的判断
- CEO / 机构负责人：快速建立机构级项目推进、风险升级和机会形成的全局认知

核心原则：

- 总结是入口，预测是价值
- 不增加员工填报负担
- 先按事件线聚合，再挂项目 / 模块 / 流程
- 短内容一段话，长内容按事件线逐段输出
- 不解释分析机理，直接给内容
- 所有模块必须共用同一套底层数据与文件层

## 1.1 底层共用原则

这套周判断要真正有价值，前提不是“多写几段分析”，而是：

`任务、日历、客户工作台、事件线、支持请求、会议、附件，必须共用同一套底层数据与文件信息。`

因此：

- 任务里上传的附件，不应只停留在任务局部
- 它应自动进入当前客户 / 项目的总文件库
- 如果任务已挂事件线，还应自动成为这条线的证据材料
- 会议纪要、支持请求、附件和任务活动要共同沉淀到同一条事件线

结论是：

- 一个项目积累的信息越多，AI 对项目判断越准确
- 一条事件线积累的信息越多，AI 对这条线的风险、机会和下一步动作判断越具体

## 2. 页面固定结构

周页面固定为四层，不再先堆事实表和任务列表：

1. `本周关键事件线`
2. `本周最值得关注的风险`
3. `本周最值得放大的机会`
4. `本周建议动作`

每一层都以事件线为最小管理颗粒，不以单条任务为最小颗粒。

## 3. 事件线摘要卡

## 3.1 卡片定位

事件线摘要卡不是任务集合卡，而是：

`一条连续事项在本周的管理判断卡`

它需要让人一眼知道：

- 这条线是什么
- 本周发生了什么
- 当前状态怎样
- 卡在哪里
- 下周最关键变化点是什么

## 3.2 固定字段

每张卡固定显示：

- `事件线名称`
- `业务分类`
- `所属项目 / 客户`
- `关联模块`
- `关联流程`
- `当前阶段`
- `本周涉及成员`
- `本周涉及任务数`
- `本周涉及会议 / 支持请求数`

## 3.3 固定五句结构

每条事件线摘要限制在 5 句以内，按固定顺序输出：

1. `这条线在推进什么`
2. `本周推进到哪`
3. `当前状态`
4. `主要阻碍`
5. `下周最关键变化点`

### 示例

`黄河基金会业务扩展线：本周重点在推进合作方案收口和业务进入点判断，已完成初版合作表达和对外材料方向梳理。当前状态从“信息摸底”进入“方案收口前的判断阶段”。主要阻碍是项目当前事项和目标对象仍不够明确，导致表述容易泛化。下周最关键变化点是尽快补齐当前事项和下一步动作，否则这条线会继续停留在浅层推进。`

## 4. 四层输出怎么组织

## 4.1 本周关键事件线

显示 3-7 条摘要卡。

排序规则：

1. 本周任务数和活动数高
2. 涉及项目级背景
3. 涉及支持请求 / 会议 / 复核链
4. 对计划或战略更重要

每条卡只回答本周推进，不提前做风险结论。

## 4.2 本周最值得关注的风险

不是把所有问题列一遍，而是只提真正升级中的线。

每条风险卡固定回答：

- 哪条事件线
- 风险正在从什么升级成什么
- 如果不处理，未来 1-2 周会怎样
- 最小干预动作是什么

### 示例

`云南连心工作坊推进线：当前风险正在从“资料补充不足”升级成“方案判断继续偏浅”。如果下周仍没有关键背景材料，这条线会继续消耗执行时间，却无法形成可判断的高质量产出。最小干预动作是先把当前事项、阻塞和下一步动作补齐，再决定是否继续推进当前方案版本。`

## 4.3 本周最值得放大的机会

机会卡只提已经形成正向势能的线，不写空泛鼓励。

每条机会卡固定回答：

- 哪条事件线
- 正向势能来自什么
- 如果继续加码，会在哪一层形成增益
- 最值得放大的最小动作是什么

## 4.4 本周建议动作

动作卡必须结构化：

- `谁`
- `在什么时候`
- `对哪条事件线`
- `做什么最小动作`

动作类型优先落到现有对象：

- 任务
- 支持请求
- 会议
- 复核
- 1v1

## 5. 角色差异

## 5.1 部门负责人视角

部门负责人优先看：

- 哪些事件线在真实推进部门计划
- 哪些事件线卡在协作、复核、流程或分工
- 哪些成员正在被哪条线持续挤压
- 哪些线需要转派、拆分、前移或补支持

部门负责人页面里，每条事件线卡应额外显示：

- `涉及岗位`
- `当前负荷信号`
- `是否偏离部门计划`
- `是否需要负责人介入`

部门负责人不需要看到机构级抽象大词，而需要看到：

- 这条线该不该我介入
- 我该先动哪里

## 5.2 CEO / 机构负责人视角

CEO 只看：

- 这条线是否在推动机构战略
- 哪些事件线是关键项目推进
- 哪些问题已经从局部事件变成组织性问题
- 哪些机会值得机构级加码

CEO 视角硬规则：

- 不输出 `部门对齐度`
- 不用“向上对齐”语言
- 不先讲机理
- 每条事件线单独成段

CEO 页面里，每条事件线卡应额外显示：

- `战略关联`
- `项目当前阶段`
- `组织级风险等级`
- `是否需要跨部门 / CEO 介入`

## 6. 自动抽取逻辑

## 6.1 聚合顺序

周判断必须按这个顺序聚合：

1. `事件线`
2. `项目 / 客户`
3. `模块`
4. `流程`
5. `单条任务`

有事件线时，不允许直接退回按单任务罗列。

## 6.2 每条事件线的输入材料

每条事件线自动吸收：

- 本周关联任务
- 本周任务状态变化
- 本周日历时间安排痕迹
- 本周会议与纪要
- 本周支持请求
- 本周复核 / 退回
- 本周附件
- 当前项目背景包
- 当前计划对象

其中“本周附件”来自统一文件层，而不是局部页面缓存：

- 任务上传附件
- 项目 / 客户总文件库
- 会议纪要附件
- 支持请求附件

## 6.3 自动摘要流水线

### 第一步：事实归并

把同一条事件线下的本周证据归并成统一事实：

- 本周做了哪些关键动作
- 哪些动作完成
- 哪些动作停滞
- 哪些动作改期 / 反复调整

### 第二步：背景挂接

给事件线自动挂接：

- 所属项目
- 当前业务分类
- 当前阶段
- 当前事项
- 当前阻塞
- 下一步动作
- 关联部门计划
- 关联机构重点

### 第三步：抽出五句摘要骨架

用统一槽位抽出：

- `what`: 这条线在推进什么
- `progress`: 本周推进到哪
- `status`: 当前状态
- `blocker`: 主要阻碍
- `next_change`: 下周最关键变化点

### 第四步：转成风险 / 机会 / 动作

根据同一套事件线骨架再派生：

- 风险卡
- 机会卡
- 动作卡

而不是重新从头分析一遍。

### 第五步：回写统一证据层

由周判断触发的后续动作，也要继续沉淀回同一条事件线：

- 新建任务
- 支持请求
- 会议草稿 / 纪要
- 新上传附件

这样“周判断”不是一次性文本，而是不断增强事件线记忆密度的过程。

## 7. 每条摘要从哪里抽

## 7.1 “这条线在推进什么”

优先级：

1. 事件线 `summary / intent`
2. 项目 `当前事项`
3. 本周任务标题共性

## 7.2 “本周推进到哪”

优先级：

1. 已完成 / 推进中的任务事实
2. 最近会议结论
3. 最近进展字段

## 7.3 “当前状态”

从以下信号综合：

- 任务状态
- 是否进入日历
- 是否待复核
- 是否跨部门
- 支持请求是否打开
- 当前阶段

状态建议只允许落入有限集合：

- `推进中`
- `收口中`
- `待确认`
- `阻塞中`
- `待补背景`
- `待决策`

## 7.4 “主要阻碍”

优先级：

1. 项目 `当前阻塞`
2. 支持请求
3. 复核 / 协作 / 改期信号
4. 轻量复盘口径

## 7.5 “下周最关键变化点”

优先级：

1. 项目 `下一步动作`
2. 事件线 `nextStep`
3. 本周未完成但高相关的连续事项

## 8. 输出规则

## 8.1 短内容

如果只有 1-2 条事件线，允许每条一段话输出，不拆子弹点。

## 8.2 长内容

如果事件线较多，必须：

- 一条事件线一段
- 一条风险一段
- 一条机会一段
- 一条动作一段

禁止把多条事件线混成一大段总评。

## 8.3 禁止项

禁止：

- 任务流水账式罗列
- 没有事件线名称的抽象总评
- 不带项目语境的“协作 / AI / 效率”空话
- 不带动作对象的泛泛建议
- CEO 视角里出现 `部门对齐度`

## 9. 实施顺序

### P1

- 周页面顶部改成四层：
  - 本周关键事件线
  - 风险
  - 机会
  - 动作
- 每条事件线摘要卡固定五句槽位
- 任务附件上传后自动归到客户 / 项目总文件库，并补一条事件线证据引用

### P2

- 部门负责人视角和 CEO 视角正式分叉
- 事件线卡补 `项目 / 模块 / 流程 / 计划关联`
- 风险 / 机会 / 动作都从事件线骨架派生
- 周判断正文优先读取统一文件层中的附件 / 纪要 / 支持请求，而不是只看任务标题

### P3

- 支持事件线 PDF 证据线导出
- 支持从事件线卡直接打开：
  - 任务
  - 会议
  - 支持请求
  - 附件
- 跨周趋势进入事件线摘要骨架
- 支持把事件线完整证据流导出成管理汇报和外部过程证据
