## 三、旧通道清单(P0 必须改)

### 3.1 工作台问答 · workspace_chat_multipass(687 行)

**现状 prompt**:
```
"你是为咨询顾问做提纲的助手. 任务: 基于客户背景包 + 战略素材 +
 已命中证据摘要, 为客户工作台的一个深度问答出大纲."
```

**问题**:
- 不读 `contract_structures` → 用户问"这个合同约定什么", 不知道
- 不读 `historical_reference_links` → 用户问"上次合同里说的那个", 找不到
- 不读 `data_gaps` → 系统不知道自己缺什么
- 不读 `approval_queue` → 不显示待审批
- 没用 `company_brain_qa` 的 8 类 evidence pack

**应改**: 调 `company_brain_qa.answer_question(client_id, question)` 替代 LLM 自己拼 prompt。

### 3.2 战略陪伴 6 段 · narrative_collector

**现状用表** (从代码 grep):
```sql
FROM v2_chunks vc
FROM atomic_facts
FROM event_lines / event_line_activities
FROM client_glossary / glossary_attributes / glossary_relations
FROM risk_signals
FROM evidence_cards (旧表, 不是 external_evidence_cards)
FROM entities
```

**问题**:
- 不读 `contract_structures` → 6 段叙事不知道合同结构
- 不读 `historical_reference_links` → 不知道复盘提的"5 月补充协议"
- 不读 `file_identities` → 不知道文件身份
- 不读 `data_gaps` → 不展示"我缺什么"
- 不读 `external_evidence_cards` (新表,与 evidence_cards 旧表不同)
- 用 client_glossary (V2.2 旧 schema), 不用 V2.3 source_registry

### 3.3 智能文件导入 · smart_file_import

**问题**:
- 调用 IngestPipeline 入库 ✓ (so atomic_facts 派生器跑)
- **不调用 file_identity_classifier** → 文件类型/角色/合同结构没识别
- **不调用 contract_structure_parser** → 合同甲乙方/期限/责任没抽
- 用户上传合同, 系统只切片不解构

### 3.4 周复盘 · review_narrative

**问题**:
- 不调 `historical_material_resolver` → 复盘提"5 月合同", 系统没尝试关联
- 周复盘的"任务汇总"和"客户故事网"没接通
