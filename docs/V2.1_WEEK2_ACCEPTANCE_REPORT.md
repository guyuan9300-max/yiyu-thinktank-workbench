## 现在能"做到"哪些 v1.0 做不到的事

1. **新代码强制走 SSOT**:
   - 后端任意业务码,凡是想知道"乐乐在哪个部门" → 必须 `directory.get_user_by_id("emp_efdd076c31").department_id`,不能再访问 `cloud_session_user.departmentName`
   - lint 在 CI 上跑,违反就拒绝合并
2. **业务码不能跨模块挖内部**:
   - `from app.modules.organization.repository import OrganizationDirectory` 会被 linter 拒
   - 必须 `from app.modules.organization import OrganizationDirectory`(从 `__init__.py`)
3. **LLM 调用全持久化**:
   - 每次 LLM 调用 → `log_prompt(db, context=..., output_text=..., duration_ms=...)`
   - 7 天后想看"上次 narrative 写错的那个客户当时 LLM 看到了什么"→ `inspector.last_for_client(client_id, intent="narrative")`
4. **冻结客户彻底不进 LLM**:
   - `ContextComposer._compose_narrative` 走 `v_active_clients` view,frozen client 直接没数据
   - 这是 v1.0 "冻结但还偶尔出现在叙事里"的根治
5. **prompt 数据驱动改进**:
   - `inspector.stats_by_intent()` 一行返回每个 intent 的成功率 / 平均耗时 / 平均 tokens
   - 评分 `update_score()` 可人工 / 自动打分,积累训练信号
