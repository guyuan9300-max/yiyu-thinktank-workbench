# 当前软件界面与功能现状盘点

更新时间：2026-03-28  
适用对象：产品、研发、运营内部盘点  
文档口径：只描述当前代码中已经存在的界面与功能，不把规划当成现状

## 状态标记

- `已完成`：前台入口与后端链路都已接通，能作为正式能力使用
- `部分完成`：已有入口和部分能力，但仍有明显限制、缺口或高频边界
- `未接通/未发现`：当前代码里没有发现完整入口或可用链路
- `仅内部底盘，前台未收口`：后端或模型对象存在，但前台还没有形成稳定可用界面

## 1. 产品总览

### 1.1 软件定位

当前软件是一个围绕中小组织经营协作搭建的桌面工作台，核心由七个一级页面构成：

1. 任务与日程
2. 客户工作台
3. 战略陪伴
4. 资讯情报站
5. 测试工作台
6. 成长手册
7. 系统设置

它的底盘不是单纯任务管理，而是把任务、事件线、客户项目、知识资料、会议、成长信号和组织治理配置接到同一套对象模型里。

### 1.2 一级导航总表

| --- | --- | --- |
| 任务与日程 | 任务收件、任务列表、内部日历、周复盘 | 已完成 |
| 客户工作台 | 项目创建、资料导入、客户问答、会议、目标、项目上下文 | 部分完成 |
| 战略陪伴 | 客户经营判断总览、战略线、组织健康、周会清单、证据底稿 | 部分完成 |
| 资讯情报站 | 选题雷达、资讯候选、洞察问答、转任务 | 已完成 |

| 成长手册 | 成长总览、XP 账本、推荐、学习工作台、沉淀条目 | 部分完成 |

### 1.3 当前整体完成度判断

- 主工作流最完整的是：`注册/登录 -> 任务协作 -> 周复盘 -> 资讯转任务 -> 基础成长记录`
- 正在往更深一层演进的是：`事件线驱动判断`、`客户项目上下文`、`战略陪伴`、`成长复盘`
- 明显还没有完全收口的是：`外部日历同步`、`细粒度文档协同编辑`、`飞书接入的公网部署条件`

## 3. 逐页功能盘点

## 3.1 任务与日程

### 页面作用

这页是软件的主执行台，负责承接协作任务、个人任务、内部日历和周复盘。

### 主要子区域/子标签

- 协作收件箱
- 任务列表
- 我的月历
- 周复盘

### 对象模型

- `任务`：最小执行对象
- `事件线`：同类、连续、跨任务的推进线
- `项目`：任务所在客户项目
- `项目模块 / 流程`：任务更细颗粒的归属
- `周复盘`：组织复盘与成长复盘
- `协作任务`：带协作者、审批流、接受/退回状态的任务

### 核心功能

1. 新建、编辑、删除任务
2. 协作收件箱接收、退回、复核任务
3. 按正式视图、清单、标签、时间等条件查看任务
4. 任务卡可展开查看说明、AI 洞察、协作状态
5. 支持事件线、项目模块、流程的归属建议与补挂
6. 月历/周历查看任务时间分布
7. 周复盘采集、历史查看、AI 判断与事件线摘要

### 依赖的数据/上下文

- 任务板与任务活动流
- 云端协作者、审批状态、mention 候选
- 事件线对象与事件线详情
- 项目上下文、项目模块、流程
- 组织模型配置
- 周复盘 dashboard / history

### 当前限制与已知问题

- 外部日历同步未见明确实现，当前主要是内部日历
- 月历拖拽仍基于 HTML5 原生拖放，虽然入口已收紧，但不是完全自定义指针拖拽
- 老任务历史上如果没挂事件线/模块/流程，AI 洞察会偏少，只是不会再假装很深
- 周判断已经接入事件线摘要、风险、机会与缺口，但时间序列预测仍偏规则化

## 3.2 客户工作台

### 页面作用

这页负责把“新客户/新项目”从一个空壳对象变成可被问答、任务、会议和学习系统共同消费的项目上下文。

### 主要子区域/子视图

- 项目列表
- 项目资料导入引导
- 项目问答工作区
- 会议流
- 目标地图
- DNA/候选文档面板
- 项目模块 / 项目流程

### 对象模型

- `client/project`：当前前台统一按“项目”使用，底层仍复用 client 主对象
- `client_dna_documents`：组织介绍、项目介绍、团队介绍、市场背景等项目背景卡
- `workspace imports / document cards / chunks`：导入资料底盘
- `meeting`：会议对象、纪要、抽取结果
- `project module / project flow`：项目级长期模块和流程

### 核心功能

1. 创建项目，进入资料导入引导页
2. 导入 Word、PDF、Markdown、PPT、纪要、方案等原始资料
3. 自动触发候选背景文档生成与知识加工
4. 手动上传 Markdown 替换项目背景卡
5. 项目工作台问答、从回答导出文档、模板填充
6. 会议创建、纪要导入、抽取、发布
7. 项目模块与项目流程的新增、挂接和任务引用

### 依赖的数据/上下文

- 导入任务、知识作业、document cards、chunks
- client DNA modules
- chat thread / messages
- meetings / goals
- project modules / flows

### 当前限制与已知问题

- “自动生成 4 张候选 Markdown”已接通，但当前更像候选稿，不是页内完整富编辑器
- 机构介绍和组织介绍没有被拆成两个独立前台模块，当前按 4 张卡收口
- 模块与流程已经有模型和入口，但维护面板仍偏轻，不是成熟的流程中心
- 客户工作台是全系统重要底盘，但部分文案和操作还在往“导入资料优先”的新路径收口

## 3.3 战略陪伴

### 页面作用

战略陪伴页是面向客户经营判断的总控台，把任务、周复盘、项目背景和会议证据压成适合陪伴式决策的视图。

### 主要子区域/子标签

- 总览
- 战略线
- 组织健康
- 周会清单
- 证据与底稿
- 资产沉淀

### 对象模型

- `StrategicCockpitSnapshot`
- `StrategicChecklistGroup / Item`
- `ReviewDashboard`
- `ClientWorkspace`

### 核心功能

1. 读取客户经营判断快照
2. 展示本周一句话、主矛盾、核心突破
3. 展示组织健康、战略线与判断准备度
4. 生成周会清单和会议包
5. 确认经营判断草案
6. 查看判断来自哪里、形成什么资产

### 依赖的数据/上下文

- 客户工作台项目上下文
- 周复盘 dashboard
- 客户任务与会议
- 战略陪伴专用接口：cockpit / checklist / meeting pack

### 当前限制与已知问题

- 前台结构完整，但它更像“战略驾驶舱第一版”，不是完全成熟的经营系统
- 依赖项目背景和事件线质量，底层信息薄时，战略陪伴的判断深度会受限
- 资产沉淀与证据底稿可看，但与成长手册、文档共享之间还没有完全统一入口

## 3.4 资讯情报站

### 页面作用

资讯情报站负责把外部资讯按雷达主题抓取、归档、提炼洞察，并转化成任务。

### 主要子区域/子视图

- 新候选
- 已保存
- 已转任务
- 候选详情侧板
- 雷达编辑与雷达辅助

### 对象模型

- `TopicRadar`
- `TopicCandidate`
- `TopicCandidateInsight`
- 本地偏好状态：saved/note/tags/read

### 核心功能

1. 新建/编辑雷达
2. 抓取/导入资讯候选
3. 对单篇候选做洞察提炼、问答追问
4. 保存本地笔记、标签、阅读状态
5. 把资讯候选转成任务
6. 对接任务列表和工作流

### 依赖的数据/上下文

- topics settings
- radar / candidate / candidate insight
- mention candidates
- task creation API

### 当前限制与已知问题

- 本地保存状态与正式后端状态混合存在，已读/标签/保存偏好部分依赖 localStorage
- 资讯到任务的转化能力可用，但更强的“运营级情报流水线”仍偏轻量

## 3.5 测试工作台

### 页面作用

测试工作台是内部分析与诊断运行区，当前承接筹款分析、系统诊断、风险 DNA 和学习沉淀入口。

### 主要子区域/子视图

- 分析模板与运行列表
- 诊断输入区
- 诊断结果与洞察
- Bettafish 外部信号整合
- 学习卡沉淀入口

### 对象模型

- `AnalysisTemplate`
- `AnalysisRun`
- `DiagnosisProfileRecord`
- `OrganizationRiskDnaDocument`
- `FundraisingKnowledgeDocument`

### 核心功能

1. 选择模板运行分析
2. 根据输入推断 workspace / audience / mode
3. 查看分析结果与结构化洞察
4. 结合组织风险 DNA 和筹款知识库做判断
5. 将结果沉淀为成长学习卡

### 依赖的数据/上下文

- analysis tools API
- settings/analysis-workbench
- diagnosis profiles
- organization risk DNA
- handbook save learning entry

### 当前限制与已知问题

- 它仍是“测试工作台”，说明产品定位上是内部实验性页面
- 模板运行和诊断能力是可用的，但统一成面向一线同事的稳定前台还没收口
- 部分能力依赖特定模板启用和组织画像配置

## 3.6 成长手册

### 页面作用

成长手册负责聚合个人成长相关的 XP、能力、推荐、沉淀和成长复盘结果。

### 主要子区域/子标签

- 总览
- 记录
- 学习
- 地图
- XP 账本抽屉
- 徽章墙
- 成长工作台

### 对象模型

- `GrowthOverview`
- `GrowthWorkbenchSnapshot`
- `LearningRecommendation`
- `HandbookEntry`
- `GrowthPendingCapture`

### 核心功能

1. 查看 XP、段位、能力卡
2. 查看最近 XP 掉落与账本明细
3. 接受/忽略成长推荐
4. 把分析结果或任务复盘转成沉淀条目
5. 查看成长工作台和资产库
6. 从成长上下文跳回任务、项目、事件线

### 依赖的数据/上下文

- growth overview / workbench / ledger / badges / recommendations
- handbook entries

### 当前限制与已知问题

- 成长复盘不是按“标签名叫私人”过滤，而是按 `tag.scope === self` 过滤
- 默认并没有预置一个自然好用的“私人/成长”自有标签，导致空状态较常见
- 成长引擎、账本、推荐底层已经存在，但产品入口仍需继续收口

## 4. 协同与账号能力专题

## 4.1 注册、登录、审批、会话保持

### 当前状态

- 注册：`已完成`
- 登录：`已完成`
- 审批：`已完成`
- 会话保持：`已完成`

### 现状说明

- 同事注册走云端员工账号体系，支持部门邀请码锁定组织与部门
- 注册后进入管理员审批，审批通过才成为正式员工账号
- 本地桌面端 auth 不是自造，而是代理云端 auth
- 本地保存 refresh token，并在云端短暂不可达时尽量保持已登录状态

### 已知边界

- 产品层仍需要继续收口“注册后岗位自填”和组织结构默认联动体验

## 4.2 日历：内部日历 vs 外部同步

### 当前状态

- 内部日历：`已完成`
- 外部日历同步：`未接通/未发现`

### 现状说明

- 内部已经有月历、周历、任务时间块、待排任务池
- 任务可进入内部日历视图并按时间排序、展示、拖排

### 已知边界

- 未查到 Google Calendar、Outlook、ICS 的清晰双向同步能力
- 因此当前不要把它描述成“外部日历同步系统”

## 4.3 任务协作

### 当前状态

- `已完成`

### 现状说明

- 有协作者模型
- 有协作收件箱
- 有接受/退回
- 有复核通过/退回
- 有任务活动流
- 有 mention 候选与支持请求

### 已知边界

- 体验层还在持续优化，例如旧任务归属建议、事件线澄清等
- 但基础协作链路本身是可用的

## 4.4 文档共享 / 知识库

### 当前状态

- 共享知识库：`已完成`
- 细粒度协作文档：`未接通/未发现`

### 现状说明

- 系统有 documents、knowledge documents、versions、chunks、document cards、DNA documents 等完整知识底盘
- 客户工作台会把导入资料转为可检索、可问答、可生成候选上下文的知识库

- 当前更像“共享知识资料库”，不是“协作文档平台”

## 6. 附录

## 6.1 关键术语

- `任务`：最小执行对象
- `事件线`：同类、连续、跨任务推进的一条线
- `项目`：当前前台面向客户的主对象
- `项目模块`：项目里的长期业务模块
- `项目流程`：模块下的标准流程
- `组织复盘`：面向组织和公共任务的周复盘
- `成长复盘`：面向 self 作用域内容的成长周复盘
- `DNA 文档`：组织、项目、团队、市场等背景说明文档
- `战略陪伴`：对客户经营状态做高层判断的驾驶舱

## 6.3 当前总体判断

这套软件已经不是单点原型，而是一套有正式账号体系、任务协作底盘、客户项目上下文、资讯转任务、成长信号和治理后台的桌面工作台。  
但它仍明显处在“主链路可用、深层协同与少数页面仍在持续收口”的阶段。内部盘点时，建议优先把它判断为：

- 执行主链：`可用`
- 管理与判断层：`可用但仍在深化`
- 外部生态同步与协作文档：`尚未完成`
