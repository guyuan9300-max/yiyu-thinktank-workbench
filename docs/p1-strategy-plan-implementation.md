# P1 实施稿：组织搭建中心承载战略与季度计划

> 对齐说明（2026-03-21）：
> 本文档继续有效，但现在从属于更高一级的总路线图：
> [AI 背景优先总规划](./ai-context-first-rollout-plan.md)
> 与 [全局模块改造清单与阶段路线图](./global-module-refactor-roadmap.md)。
> 后续执行时，以“字段先服务 AI 思考背景、自动生成优先、人工轻确认、P1 只做低频根语境母盘”为准。

## 目标

在不进入任务与日历主链路的前提下，先把“上游计划骨架”落到组织搭建中心。

P1 只做三层：

1. 年度战略
2. 当前季度机构计划
3. 部门季度计划

P1 不做：

1. 部门月度计划
2. 从计划自动拆任务
3. 从计划自动排日历
4. 任务页内的计划入口

## 一、P1 页面定位

P1 不新开一个孤立大模块，直接在：

- `系统设置`
- `组织搭建中心`

里新增一个明确分区：

**战略与计划**

它的职责是：

- 承载机构年度目标
- 承载当前季度机构计划
- 承载每个部门的季度计划
- 为 P2 的任务与日历承接层提供上游上下文

## 二、页面结构建议

组织搭建中心新增一个二级结构：

1. 组织骨架
   - 组织负责人
   - 部门
   - 岗位
   - 成员

2. 战略与计划
   - 年度战略
   - 当前季度机构计划
   - 部门季度计划

3. 流程与权限
   - 流程模板
   - 任务控制规则

也就是说：

- 组织骨架解决“谁在组织里”
- 战略与计划解决“这个季度组织和部门在做什么”
- 流程与权限解决“怎么运转”

## 三、P1 页面区块

### 区块 A：年度战略

放在“战略与计划”顶部。

显示内容：

- 年度目标
- 年度叙事
- 成功定义
- 年度关键指标
- 年度风险

编辑方式：

- 右侧详情表单
- 或顶部整卡编辑区

交互要求：

- 未填写时显示“待补年度战略”
- 填完后显示完整度和最后更新时间

### 区块 B：当前季度机构计划

放在年度战略下方。

显示内容：

- 当前季度
- 季度主题
- 季度目标
- 成功标准
- 关键押注
- 主要风险

交互要求：

- 如果没有年度战略，仍允许填写，但要提示“建议先补年度战略”
- 当前季度卡是所有部门计划的上游母卡

### 区块 C：部门季度计划

放在部门卡片层，不再只是部门介绍。

每个部门卡至少显示：

- 部门名称
- 负责人
- 部门职责
- 当前季度目标
- 当前季度关键产出
- 当前季度成功标准
- 当前季度关键动作数
- 当前状态标签

状态标签建议：

- `待补季度计划`
- `计划中`
- `稳定`
- `风险待处理`

右侧详情编辑区增加字段：

- 部门职责
- 当前季度目标
- 当前季度关键产出
- 当前季度成功标准
- 当前季度关键动作
- 当前季度依赖
- 当前季度风险

## 四、字段最小集

P1 先只落最小必填集，避免页面过重。

### 年度战略最小集

- `annual_objective`
- `annual_narrative`
- `success_definition`

### 当前季度机构计划最小集

- `quarter`
- `theme`
- `quarter_goal`
- `success_criteria`

### 部门季度计划最小集

- `department_id`
- `quarter_goal`
- `key_deliverables`
- `success_metrics`

P1 先不强制：

- 复杂指标
- 多负责人
- 多季度切换器
- 历史归档

## 五、完成度计算建议

当前组织搭建中心已经有完整度概念，P1 可以直接接进去。

### 年度战略完成度

- 年度目标已填：30%
- 年度叙事已填：30%
- 成功定义已填：40%

### 当前季度机构计划完成度

- 季度主题已填：20%
- 季度目标已填：35%
- 成功标准已填：45%

### 部门季度计划完成度

- 部门职责已填：20%
- 季度目标已填：30%
- 关键产出已填：25%
- 成功标准已填：25%

部门卡上的“80% / 60%”就不再只是介绍完整度，而是计划完整度。

## 六、P1 页面文案方向

这个区域不能写成“组织介绍管理”。

应该写成：

- “战略与计划”
- “当前季度组织计划”
- “部门季度计划”
- “为后续任务、日历、成长判断提供组织上下文”

文案重点是：

- 这是运行中的计划，不是资料归档
- 后续任务与建议系统都会读取它

## 七、P1 的 AI 使用方式

P1 阶段 AI 的职责不是生成任务，而是：

1. 帮助用户从资料中整理年度目标
2. 帮助把季度主题收成结构化字段
3. 帮助从部门职责中抽取季度目标草稿
4. 对缺失项给出待补提示

P1 阶段 AI 不做：

- 自动拆月度计划
- 自动生成任务包
- 自动排会

## 八、与其他模块的关系

### 成长系统

P1 就可以开始读取：

- 当前部门季度目标
- 当前机构季度目标

即使还没有月度计划，也能先做比现在更精准的解释层。

### 任务与日历

P1 不改主界面，只做“未来可接入”的上游准备。

### 全局助理

P1 可记录待补缺口，但不正式上线全局助理。

## 九、P1 验收标准

做到以下 5 点，就算 P1 成立：

1. 组织搭建中心能录入年度战略
2. 能录入当前季度机构计划
3. 每个部门卡能录入季度计划
4. 卡片完整度能反映计划字段，而不是只反映介绍字段
5. 成长和后续 AI 读取接口能拿到这三层结构

## 十、P1 实施顺序

1. 先补数据结构
   - 年度战略
   - 当前季度机构计划
   - 部门季度计划

2. 再补组织搭建中心 UI
   - 新增“战略与计划”分区
   - 改部门卡
   - 改右侧详情区

3. 再补完整度逻辑

4. 最后补读取接口
   - 给成长系统
   - 给后续 AI 上下文

## 十一、P2 接口预留

虽然 P1 不做任务页，但字段设计要提前给 P2 留口：

- `department_quarter_plan.id`
- `department_id`
- `quarter`
- `quarter_goal`

因为 P2 的部门月度计划、任务绑定、日历绑定都会往这层挂。
