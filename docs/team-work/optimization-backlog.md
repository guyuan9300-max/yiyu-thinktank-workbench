# 益语智库平台 — 全面优化清单

基于 Reality Audit 的代码审计结果，按四条主线分类。

---

## 主线 A：写入能力（数据进入记忆系统的通畅度）

| # | 问题 | 当前状态 | 影响 | 优先级 |
|---|------|---------|------|--------|
| A1 | 任务创建/更新不触发 memory_facts 写入 | 只写 growth_signal，不写核心记忆 | 每天最高频的操作不推动记忆积累 | P0 |
| A2 | 选题雷达完全不写入记忆 | 选题发现→任务，但发现过程本身不沉淀 | 情报洞察丢失 | P1 |
| A3 | 客户对话（chat）不写入结构化记忆 | 549条对话消息只在 chat_messages 里 | AI 对话中的洞察没有进入认知池 | P1 |
| A4 | 手册条目创建不触发 notebook/event_line 刷新 | 只写 growth_signal | 经验沉淀不推动项目认知更新 | P1 |
| A5 | 没有自动定期记忆刷新 | 全部事件驱动，无后台定时任务 | 如果用户不做特定操作，记忆停滞 | P1 |
| A6 | local_memory（文件）和 DB memory 两套独立系统 | 不交叉引用 | 信息孤岛，dream_cycle 只整理文件不触及 DB | P2 |
| A7 | 录音/语音沉淀通道缺失 | 会议支持文字纪要但无语音转文字 | 大量面对面/电话沟通无法进入系统 | P2 |

---

## 主线 B：理解能力（系统是否真正理解了数据）

| # | 问题 | 当前状态 | 影响 | 优先级 |
|---|------|---------|------|--------|
| B1 | understanding_builder 未接入任何生产链路 | 完整实现但只在测试中调用 | 系统有理解能力但从不展示 | **P0（Round 1 解决）** |
| B2 | 战略陪伴页全用 mock 数据 | 后端有 build_strategic_cockpit_snapshot 但前端不调 | 用户看到的是假数据不是真实理解 | P0（Round 2） |
| B3 | memory_facts 写了 273 条但几乎不被读回 | 只有 gather_project_context_for_ai 在读 | 记忆写了等于没写 | P0 |
| B4 | 任务详情页不展示系统理解 | 只显示用户输入的标题和描述 | 系统像记事本不像助理 | P0（Round 1 解决） |
| B5 | 客户工作台有 notebook 数据但不展示为"理解" | 数据在 DB 里，前端不渲染 | 用户不知道系统了解客户多少 | P1 |
| B6 | 事件线记忆有 8 份快照但不被用于上下文增强 | 快照维护了但不回流到任务/研判 | 事件线是记录不是关系记忆 | P1 |
| B7 | 周复盘分析很深但结果不回流到下周任务 | review_analysis 算了 15+ 维度的分析 | 每周的理解不会影响下周的工作 | P1 |

---

## 主线 C：经验升级能力（记忆是否能升级为方法）

| # | 问题 | 当前状态 | 影响 | 优先级 |
|---|------|---------|------|--------|
| C1 | 多次复盘不能自动升级成方法/SOP | 复盘数据存了但不做跨期聚合 | 反复犯同样的错 | P1 |
| C2 | 成长信号只在成长中心展示，不回流到工作流 | XP/徽章独立于任务系统 | 成长是最后一个独立模块，不是贯穿的 | P1 |
| C3 | handbook_entries 只有 5 条且全是手动创建 | 没有自动从复盘/会议中提取经验条目 | 经验墙靠人工填，注定空洞 | P1 |
| C4 | local_memory 的 dream_cycle 不触发经验升级 | dream 只整理文件大小，不提取模式 | 记忆整理了但没变聪明 | P2 |
| C5 | 模板/SOP/检查清单系统不存在 | 只有 project_modules 和 project_flows | 经验无法固化为可复用的流程资产 | P2 |

---

## 主线 D：反向赋能能力（认知池是否反过来帮用户）

| # | 问题 | 当前状态 | 影响 | 优先级 |
|---|------|---------|------|--------|
| D1 | 创建任务时没有智能提示/上下文预填 | 空白输入框 | 用户每次从零开始 | P1 |
| D2 | 客户工作台的聊天框没有注入项目记忆 | chat 是通用对话 | AI 不了解当前客户背景 | P1 |
| D3 | 会前准备不使用事件线记忆 | 会议创建是空白的 | 每次开会都要重新介绍背景 | P1 |
| D4 | 周复盘模板不使用上周分析结果 | 每周从空白开始写 | 复盘不能基于上周的判断继续深入 | P1 |
| D5 | smart_brief 系统存在但覆盖有限 | 只在部分任务上生成 | 大部分任务没有智能提示 | P2 |
| D6 | 组织简报/周报不基于真实记忆生成 | review_narrative 生成但不稳定使用 | "懂业务的助理"感知弱 | P2 |

## 跨主线的基础设施问题

| # | 问题 | 当前状态 | 影响 | 优先级 |
|---|------|---------|------|--------|
| X1 | 战略陪伴页用 mock 数据 | 全部硬编码，不调 API | 用户看到假的"大脑" | P0 |
| X2 | 任务中有脏数据（重复嵌套命名） | 如"继续推进「继续推进「..." | 影响所有分析质量 | P0 |
| X3 | 6 场会议全停在 prepared 阶段 | 无一完成全流程 | 会议这条记忆管道实际上从未跑通 | P1 |
| X4 | W14 周复盘内容为空 | 创建了但没写内容 | 复盘管道有但用户没用起来 | P1 |
| X5 | 手机端存在但未与桌面端打通 | mobile/ 目录有代码但独立 | 移动场景无法贡献记忆 | P2 |

## 建议的 Sprint 顺序

| Round | 主题 | 解决的问题 |
|-------|------|-----------|
| **1** | 接入 understanding_builder 到任务详情 | B1, B4 |
| **2** | 战略陪伴页接入真实数据 | X1, B2 |
| **3** | 任务创建/更新触发 memory_facts 写入 | A1, B3 |
| **4** | 成长信号回流到任务提示 | C2, D1 |
| **5** | 周复盘分析结果注入下周上下文 | B7, D4 |
| **6** | 客户聊天注入项目记忆 | A3, D2 |
| **7** | 自动从复盘中提取经验条目 | C3 |
| **8** | 事件线记忆回流到任务和研判 | B6 |
| **9** | 会前准备使用事件线记忆 | D3 |
| **10** | 后台定期记忆刷新 | A5 |
