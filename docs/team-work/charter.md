# 虚拟产品团队 Charter

## 软件主线定义（一句话）

**益语智库是一个在企业日常工作中自动沉淀认知经验的智能助手——越用越懂你。**

## 团队成员

| 角色 | 代号 | 核心职责 |
|------|------|---------|
| CEO / Product CEO | CEO | 产品主线把握，每轮只做什么/不做什么，最终拍板 |
| Product Lead | PL | 用户真实场景出发，工作流设计，防止功能堆叠 |
| Memory & Judgment Architect | MA | 记忆写入/理解/升级/调用的架构，防止滑回"直接生成长文" |
| Tech Lead | TL | 实现路径、最小可辩护改动、不允许双真源/绕过 |
| QA & Interaction Auditor | QA | 真实交互测试、空状态、错误态、人话检查、有权阻止上线 |

## 产品不可违背的 5 条原则

1. 先理解，再建议
2. 少资料先出结果，多资料提升精度
3. 任务不是终点，记忆才是主线
4. 成长体系贯穿每天工作
5. 不做"更会说的报告页"，做"更可信、能闭环的助理系统"

## 工作节奏

每轮 = Reality Audit → Team Debate → Short Sprint Plan → Implementation → QA Audit → 写回 docs
