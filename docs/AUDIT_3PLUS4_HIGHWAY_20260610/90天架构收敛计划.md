# 90 天架构收敛计划

> 接续 30 天主干修通。30 天解「地基没浇/写入漏水/无回归网」的急；90 天解「蜘蛛网结构性收敛」——让益语智库从「一堆功能 + 半成主干」真正收敛成「以数据中心为地基、算法层为调度、应用层为入口出口」的信息高速公路。原则：不重写，按职责切分既有巨石，逐块迁移有回归网兜底。

## 收敛一 · 统一算法层调度（取材 + 生成 + 溯源 三条干道收口）

- **取材收口**：把下一步雷达、智能填表等绕开 `retrieve_knowledge_bundle` 的内联直查（D2-04）全部迁到共享检索干道；让「四仓统一检索」名副其实（当前只有富化单仓文档干道）。
- **生成收口**：3 个硬编码 Ollama 模块（D2-05）全接入 AiService；提供共享本地-LLM 客户端 + 强制护栏，杜绝「谁要本地模型就手搓一份」。
- **context builder 收口**：要么把死原型 `ContextComposer`（D2-01）做成真正多 intent 的统一装配器并强制全链路接入，要么彻底删除它消除虚假架构信号——二选一，不留半成竞争者。
- **溯源台账收口**：让所有 AI 产出走统一 `reasoning_traces`（D2-07），消灭各生成器平行 schema；prompt 进中央注册表（D2-08，散落 18+ 文件收一处）。

## 收敛二 · 拆三块巨石（有回归网后逐块迁移）

- **main.py 60258 行 / 630 路由**：按业务域（任务/客户/战略/资讯/成长/数据中心/同步）切成 router 模块，迁出 `backend/app/api/`（当前近空仅 deprecated）。每迁一组路由跑一遍 §主干道一 的冒烟测试。
- **App.tsx 30885 行 / 96 组件**：按页面拆组件文件，state 下沉到各页面 hook。
- **models.py 9244 行 / 734 类**：按域拆 schema 包。
- **Repository 真落地（D6-02）**：把 1242 处 db.execute 直查逐域收敛到 7 个已有 *Repository（当前只 ClientRepository 真用），完成那场「停在数据层的半截迁移」。

## 收敛三 · 四仓物理收敛 + 死表清理

- **原始仓 vs 认知仓隔离（D1-07）**：AI 衍生文档（judgment/fact_card/enrichment）与原始材料物理分库或加 `evidence_tier` 硬隔离，避免「AI 加工产物被当一手证据再喂回 LLM」的回声放大；检索层按 canonical_kind 降权。
- **重复表合并**：`document_chunks`→`v2_chunks` 单轨；`exp_wall_quotes`/`handbook_entries` 经验墙双表收一张；`_cloud` 影子表机制统一。
- **死表清理（31 张）**：data_center_proposal_drafts/execution_tickets/ai_task_plans/key_decisions/goal_records 等「设计了未启用」的——按产品意图决定「接线启用」还是「DROP」，不留门面误导后续审计。`local_identities` 随 local-auth 剥离一并清。

## 收敛四 · 状态唯一真相源（SSOT）

- 文件数量 4 源（D5-06）收敛到统一 count 端点，各页面 widget 取同一口径。
- 索引状态以 v2_chunks 为准（已证近全覆盖），废弃 stale 旧向量 manifest。
- 客户状态展示从 `max(updated_at)` 改状态机（D5-08），不让 failed 被选成「最新」。
- `job_stage_runs` 事件流表正名（D5-01/02）——要么改成真状态表（就地 update），要么改名/加视图明确「这是事件日志」，消除审计者与代码的语义混淆。

## 收敛五 · 可观测性闭环

- 全链路统一 `request_id/run_id/task_id`（D6-04 当前 4 处），后端结构化日志贯通。
- 前端全局错误出口（window error/unhandledrejection/ErrorBoundary）挂兜底接线到已建成的远端上报。
- 289 处 `except: pass`（D6-06）按域审查，区分「有意忽略」与「静默吞 bug」。

## 90 天北极星验收

> 回到北极星：任一用户动作进系统后能否被追踪「从哪入口→变成什么数据→进哪仓→被哪算法调用→在哪呈现→是否有来源→能否回流」。

- [ ] 抽查 10 类入口（文件/任务/会议/对话/复盘/资讯/导入/录音/纠错/计划），每类都能在仪表盘画出完整链路，断点=0
- [ ] 四仓覆盖率指标全部达标（来源 100% / 事实回溯 ≥90% / 富化主力客户 ≥90% / 认知厚客户 ≥80%）
- [ ] 算法层三干道（取材/生成/溯源）零旁路，prompt 中央注册
- [ ] main.py < 15k 行（路由迁出）、App.tsx 按页面拆分
- [ ] 死表/重复表清零或正名；状态 SSOT 无 max() 掩盖
- [ ] 综合评分 45 → ≥70（功能能跑且架构黑箱基本消除）
