## 0. 一页结论

**益语智库现在是不是「以中央数据中心为底座、所有模块围绕统一组织记忆运转」的信息高速公路？**

**结论：还不是。它是「一条半成的主干道 + 多条各写各表的辅路 + 几座没浇地基的门面建筑」。** 主干雏形确已出现——一条文档深读流水线（导入→实体→事实→索引→叙事）、一个事实上的统一 AI 网关 `AiService`、一条共享检索干道 `retrieve_knowledge_bundle`、以及客户工作台/战略陪伴两块由后端统一信息包驱动、来源可见性很强的呈现面。但作为「高速公路地基」的四件事都没浇上：

1. **统一来源登记门面化**：蓝图设计的 `source_registry`（来源登记表）全库仅 **4 行**，2288 条事实里仅 **0.17%** 挂得上来源；最大的入口（文件导入 1433 篇）完全绕过登记；80% 的写入入口不登记。
2. **组织归属结构性为空**：documents/tasks/meetings 的 `organization_id`（组织 id）真库 **100% 为空**，`mirror_*`（组织/部门/人员镜像四表）**全 0 行**——身份中枢从未落数据，数据中心声称的「组织/部门可见性」无数据支撑。
3. **写入侧没有收口**：共识仓（事实/承诺/风险）被 **5–7 套独立写入路径**各自直插主表，无统一闸门、无唯一约束，真库已观测到 **5 组完全重复的承诺**。
4. **没有在跑测试的 CI**：90 个历史失败测试被集中静音保绿灯，而仓库唯一的 CI workflow 根本不调 pytest——回归保护近似于零。

**独立评分：45 / 100**（详见 §15）。落在「严重黑箱、不建议稳定对外发版」区间，但比 6/8 的 38 分、5/28 65-E 的 41 分略高——因为这次独立复核**纠正了上一轮几处高估**（队列「卡死 55 天」实为事件流误读、富化主索引其实近全覆盖），并**确认了几条真接通的主干**。

**计数（最终，经排+1 调整）：P0 = 2 ｜ P1 = 20 ｜ P2 = 23。** 两个 P0 都是「写入侧分裂」与「无测试 CI」这种地基级问题，不是会删数据的崩溃。

## 12. 代码结构与蜘蛛网（D6 第一部分）

- **D6-01 P1**：三块巨石坐实——`main.py` **60258 行 / 630 路由几乎全内联其内**；`App.tsx` **30885 行 / 492 个 useState·useEffect / 96 组件挤一文件**；`models.py` 9244 行 / 734 类。
- **D6-02 P1**：Repository 模式是门面——`backend/app/modules/` 有 7 个 `*Repository` 类，但 main.py 只 1 处真实例化（ClientRepository），全仓走 db.py 薄包裹器 `db.execute/db.fetchone`（**主仓 1242 处直接 execute、1331 条裸 SQL 字面量**），业务逻辑与 SQL 彻底耦合在路由函数体里，无数据访问层——这是「停在数据层、从未到达路由层的半截 Repository 迁移」（排+1 根因）。
- **D6-05 P1**：同义重复表并存（见 §6）；`mirror_users` 唯一写入口是手动 POST 端点 `/api/v1/local/organization/sync`(main.py:60056)，从未被触发 → **人物身份解析主链路在真库静默退化为全 unknown**（排+1 校正：非「死 schema 该 DROP」，是身份中枢未同步）。
- **D6-06 P2**：289 处 `except…: pass` 静默吞异常；5 个 `.DEPRECATED` 废弃文件留树。

## 19. 如果不修会带来的风险

- **不修来源登记/org 归属**：数据中心声称的「组织/部门可见性、来源可追溯」永远是空字段——多组织/权限隔离无数据支撑，对外宣称的「统一组织记忆」名不副实。
- **不修写入侧收口（P0）**：每次发版脏承诺继续扩散，「下一步/承诺」列表重复提醒，读出侧再好的质量门也被写入侧污染抵消。
- **不修测试 CI（P0）**：19 个并行 AI worktree 持续 churn 60k 行 main.py，无回归网 = 每次合并都在赌；6/8 的 6 簇 bug 正是这种环境的产物。
- **不修队列无 reaper / 成长同步**：local_model/deep_read 静默冻结、成长数据 99% 不上云，用户侧表现为「AI 不动了/手册不同步」却无健康提示。
- **不拆巨石**：60k 行 main.py 让任何新功能都得在蜘蛛网里穿线，技术债利滚利。
