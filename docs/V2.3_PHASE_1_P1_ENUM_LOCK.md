## 1 · content_role 枚举(9 类 + 3 类 V2.3 扩展)

**V2.2 已有(9 类)**:
| 值 | 含义 | 来源典型 |
|---|---|---|
| `fact` | 客观事实(可验证) | 合同 / 章程 / 官方公告 |
| `decision` | 决策记录 | 会议决议 / 用户拍板 |
| `risk` | 风险信号 | 内部预警 / 外部舆情 |
| `observation` | 观察(主观) | 用户口述判断 / AI 推断 |
| `plan` | 计划 / 意图 | 任务 / next_step |
| `lesson` | 经验 / 教训 | 复盘 / 个人手册 |
| `quote` | 引用原话 | 会议纪要 / 对话记录(必带 speaker) |
| `commitment` | 承诺 | 我方对客户 / 客户对我方 |
| `speculation` | 推测(未验证) | UGC / 自媒体 / AI 大胆推断 |

**V2.3 新增(3 类)**:
| 值 | 含义 | 来源典型 |
|---|---|---|
| `interpretive_claim` | 用户解释性主张(非事实,带价值判断) | 工作台对话:"我觉得客户真正担心的是预算" |
| `method_card` | 可复用方法(限域有效) | 成长中心个人手册 |
| `external_intel` | 外部公开情报(待验证关联) | intelligence_items / 外部新闻 |

**总计 12 个 content_role**(锁定,LLM extractor prompt 必带 enum 约束)。

## 8 · 这份枚举锁定后,谁不可以再改

- DocumentLLMExtractor prompt 提到的 content_role 必须只用 12 个
- narrative_generator 引用 atomic_facts 时按 content_role 分组渲染
- 任何新 source 进入必须填 source_type + source_channel + (可选 source_role)
- B AI 在 baseline runner / 集成测试 中按本枚举核对

**新增枚举 = 改本文档 + commit + A/B 双方知会**。
