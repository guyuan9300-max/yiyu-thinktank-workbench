## 已知 caveat

- **cloud session 失效时 W1 dim_11 / dim_12 会 SKIPPED**(不是 FAIL)。这两维需要本地 prod db 里有有效 `cloud_access_token` 和 `cloud_session_user`。session 过期时整段链路无法跑,本次 W3 顺手修了 Row 对象 truthy 误判的 robustness bug。
- **W4 必须做**:`growth_engine.py` 的 7 处 commitments SQL 是真正的 commitment 重心,本次因业务逻辑复杂(streak / trend / 月度兑现)风险大,留到 W4 用 TDD 包紧再切。
- **Repository 一致性**:`_ConnAdapter` 模式在 glossary / commitment 两个 Repository 都复制了一遍。W4 视情况抽到 `app.modules._shared`(linter 需要例外该路径)。
