# 法律文档目录（docs/legal/）

> **重要前置说明**
>
> 1. 本目录下所有文档均为**结构合规、可直接使用的工作版本**，但**正式发布前必须由具有中国法律执业资格的律师（最好同时熟悉 Apple App Store 与 EU GDPR）做一次终审**。
> 2. 文档中所有标记 `【需填写: XXX】` 的位置必须补齐主体信息（公司全称、注册地、联系邮箱、备案号等），否则法律层面不成立。
> 3. 一旦上线，**任何重大变更必须同步更新版本号 + 在 App 内推送变更告知 + 重新征得用户同意**——这是 PIPL §17 的硬性要求。

## 上线流程（建议）

1. **填空**：把所有 `【需填写: XXX】`占位符补齐。重点：公司主体名称、注册地址、客服邮箱、个人信息保护负责人邮箱。
2. **律师终审**：把补齐后的 4 份文档（用户协议 + 隐私政策 + App Store 合规清单 + 实施指南）发给律师审。预算建议留 5000-15000 元（差不多覆盖中等规模软件的合规审计）。
3. **集成到代码**：按 `04-integration-guide.md` 把文档内容落到首启弹窗、关于面板、设置页。
4. **跑 license 脚本**：`node docs/legal/scripts/collect-licenses.mjs` 自动生成 NOTICE 文件，附在 app 关于面板的"开源许可"入口。
5. **官网托管**（App Store 必需）：把 `01-user-agreement.md` 和 `02-privacy-policy.md` 转 HTML 部署到一个稳定的公网 URL，例如 `https://yiyu.org/legal/user-agreement` 和 `https://yiyu.org/legal/privacy-policy`。Apple 提审时要填这两个 URL，且必须公网可访问。
6. **备案**（中国）：根据公司主体所在地，到所在省/市的网信办做软件备案；如涉及个人信息处理，需做"个人信息保护影响评估（PIPIA）"。

## 法律基础

本套文档基于以下法规框架编写：

| 法律 | 适用 | 关键合规点 |
| --- | --- | --- |
| 中华人民共和国个人信息保护法（PIPL，2021） | 处理任何中国用户个人信息 | 单独同意、最小必要、可撤回、可删除、可携带 |
| 中华人民共和国数据安全法（2021） | 处理数据 | 数据分级分类、安全保护义务 |
| 中华人民共和国网络安全法（2017） | 网络运营者 | 实名注册、日志留存（不少于 6 个月） |
| 中华人民共和国消费者权益保护法 | C 端用户 | 知情权、选择权、公平交易权 |
| 中华人民共和国电子签名法 | 在线协议生效 | 协议条款显著提示、可获得性 |
| GDPR（欧盟通用数据保护条例） | 任何 EU 用户 | 7 项权利、72h 通报、DPO |
| CCPA（加州消费者隐私法案） | 任何加州用户 | 不歧视、出售拒绝权 |
| Apple App Review Guidelines | 上架 App Store | §5.1 隐私、§5.1.1 数据收集 |
| Apple Developer Program License Agreement | 注册 Apple 开发者 | 二次分发、SDK 使用 |
