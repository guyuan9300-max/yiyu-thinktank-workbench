## 一 · 测试核心命题

> **一个项目能不能被系统真正"想清楚"**

不是检索能力,不是抽字段能力,是 **理解能力**:

```
12 条不同来源的数据进系统
  ↓
  数据中心分型 / 抽取 / 交叉验证 / 冲突识别 / 澄清生成 / 故事网
  ↓
50 个不同问题问回去
  ↓
看回答是否带证据、不编造、能分客户事实/我方判断/外部观察、能识别冲突
```

## 五 · 50 个测试问题(10 大类 × 5 变种)

按顾源源指令 "至少 50 个,不同类型,不同长短":

```
Q1.1-1.5  · 项目来源类:    简单 / 详细 / 历史 / 对比 / 未来
Q2.1-2.5  · 预算类:        当前 / 历史 / 差异 / 外部 / 谁定
Q3.1-3.5  · 项目范围类:    当前 / 历史 / 差异 / 外部 / 原因
Q4.1-4.5  · 人物角色类:    执行 / 拍板 / 协调 / 沟通 / 角色变化
Q5.1-5.5  · 风险类:        主要 / 隐性 / 缓解 / 优先级 / 对比
Q6.1-6.5  · 承诺类:        未完成 / 已完成 / 谁承诺 / 延期 / 触发条件
Q7.1-7.5  · 冲突识别类:    识别 / 数量 / 严重度 / 来源 / 解决
Q8.1-8.5  · 时间线类:      按月 / 按事件类型 / 关键节点 / 缺口 / 未来预测
Q9.1-9.5  · 信息分类类:    客户官方 / 我方 / 外部 / 口述 / 不同维度
Q10.1-10.5 · 下一步类:     问客户 / 做事 / 补材料 / 紧急度 / 责任人
```

每题不同长短(短句 / 中长 / 复合 / 长情境)、不同类型(具体 / 开放 / 对比 / 历史 / 未来)。

## 六 · 5 维评分(100 分制)

| 维度 | 权重 | 判定 |
|---|---|---|
| D1 数据分型正确性 | 20 | source_type 命中率 ≥ 90% |
| D2 结构化抽取完整性 | 20 | 关键实体召回 ≥ 85% |
| D3 冲突发现与澄清质量 | 20 | ≥4 核心冲突 + 高质量澄清问题 |
| D4 项目故事卡质量 | 20 | 10 段全有真数据 + 新同事 5 分钟能讲清 |
| D5 50 问答与闭环 | 20 | ≥ 40/50 正确 + 不编造 + 来源标 |

**等级**:
- **A 级**: ≥85 — 可进真实客户试点
- **B 级**: 70-84 — 内测中,不可进真客户
- **C 级**: <70 — 数据中心假设未成立
