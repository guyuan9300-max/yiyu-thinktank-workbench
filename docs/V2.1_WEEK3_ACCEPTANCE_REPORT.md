## 已知 caveat

- **W1 dim_8 / dim_11 / dim_12 SKIPPED**:本地 prod db 的 `cloud_access_token` 和 `cloud_session_user` 已空(火山云 session 过期)。本次顺手修了 W1 acceptance 的 robustness bug — `sqlite3.Row` 对象不论 value 是否为空都 truthy,以前 skip 检查写成 `if not token or not url` 会进入 sync 然后才在 sync_organization_directory 里 FAIL。现在改成检查 `token["value"]` / `url["value"]`,空字符串正确 SKIP。重新登一次火山云后 3 维度即可重新跑通。
- **未切的 SQL 还剩**:5 个 glossary 服务文件 + 5 个 commitment 高密度文件 + main.py 3 处 + 7 个 P14-D 脏文件,留 W4。
- **`_ConnAdapter` 重复**:glossary / commitment 两个 Repository 各自实现一份。W4 视情况抽到 `app.modules._shared`(linter 例外该路径)。
