## 三 · LLM 答案质量样本

### 答得非常准确的题(原文摘录)

**Q06**(预算多少):
> 乡村儿童阅读陪伴项目的当前权威预算为300万元。`[fact:af_9dc64352244b]`

**Q07**(为什么 500 万还出现):
> 资料中提到的500万元是项目的旧版预算(`[fact:af_4c6ee20e042e]`),当前权威版本的总预算是300万元(`[fact:af_9dc64352244b]`)。因此,500万元这个数字现在不再作数。

**Q03**(项目调整路径):
> 乡村儿童阅读陪伴项目的最初设想是覆盖10所学校,总预算为500万元`[fact:af_e14de5f67405][fact:af_4c6ee20e042e]`。后来调整为先在3所学校进行试点,并将总预算减少到300万元`[fact:af_cbc5cee5fa87][fact:af_9dc64352244b]`,以验证机制后再考虑扩大规模。

**Q01**(项目怎么来的) — **不知道说不知道**真兑现:
> 乡村儿童阅读陪伴项目首次沟通日期为2026年5月6日,但关于该项目最早的来源信息**在提供的证据中没有详细说明**。`[fact:af_8f5b2b246a22]`

### 答错的题(评分函数严)

LLM 答案大多在表达,但 `must_contain` 关键词没字面命中:
- `Q14`(外部口径):LLM 答了"需要更新外部口径",但没写"外部口径"四字
- `Q17`(陈老师):LLM 答了"陈老师是用户口述的最终拍板人,需要客户确认",但"待确认"是用"需要客户确认"表达

## 四 · 真发现

### ✅ V2.4 数据中心 + LLM 受限问答 真兑现 4 件事

1. **Evidence 真喂得动 LLM**: 33 facts + 6 clarifications + 6 risks + 1 commitment 全部装进 prompt(~2K context),qwen2.5:14b 8K context 够用
2. **LLM 真受约束**: 100% 引用 `[fact:xxx]`、0 幻觉、不知道说不知道
3. **答案质量优秀**: 多条复杂题答得比 deterministic answerer 更好(Q03/Q07 等)
4. **速度可在线**: 平均 5s/问,50 问 4 分钟跑完

### ⚠️ 评分函数仍有问题

跟 P1-5 一样,`must_contain` 字面匹配过严,惩罚了 LLM 自然语言多样性。LLM 答了"需要客户确认"被判没命中"待确认" — 但这是同义表达。

### 🎯 P1 阶段整体结论

```
P1-5 LLM 抽取:    0 幻觉 / 12 datum 全跑通 / 速度可用
P1-6 LLM 受限问答: 0 幻觉 / 100% 引用 / 速度可用 / 答案质量优
```

**V2.4 数据中心已经具备给 LLM 当 evidence 引擎的能力。**
