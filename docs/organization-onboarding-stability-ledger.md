# 组织接入与成员管理稳定性修复台账

本轮修复遵循蓝图：云端是组织、成员、部门、任务清单的真相源；本地只同步当前工作空间镜像；前端只表达状态，不伪造云端数据。

| 问题 | 真相源 | 当前偏差 | 修复层级 | 禁止旁路 |
| --- | --- | --- | --- | --- |
| 新组织缺“收集箱”导致云端拒绝任务 | cloud_backend `task_lists` | 本地可生成清单，但云端未初始化时仍拒绝任务 | 云端幂等初始化；本地入组后同步镜像 | 不在本地伪造云端清单作为成功依据 |
| 成员刚加入后列表短暂为空/不全 | 云端员工与部门接口 | 本地 mirror 未及时同步，前端只看到空态 | 本地后端触发同步与短重试；前端显示同步中和刷新 | 不跨工作空间读取成员兜底 |
| 成员管理有时消失 | 云端员工管理接口 | 本地吞掉接口错误并返回空数组，前端按“无内容”隐藏模块 | 本地返回可理解错误；前端固定入口和状态 | 不用前端猜测权限或制造假员工 |
| 重置密码失焦/不可见 | 云端密码哈希 | 行内输入随表格重渲染失焦，且无法看清输入 | 前端稳定弹窗，增加显示/隐藏开关 | 不恢复或展示原密码，不生成临时密码 |
| 内部错误词泄露 | API 边界层 | `JSONDecodeError`、`Invalid task list`、`工作空间不存在` 等直接透出 | 本地后端与前端统一翻译 | 不只改单个按钮提示 |
