# 资讯情报站六阶段改造计划与阶段 0 原型

记录日期：2026-05-13

## 硬规则

- 本文中的样张只用于产品原型、内容合同和验收标准。
- 样张不得作为模拟内容、默认内容、fallback 内容写入情报站、数据库或前端。
- 情报站只能展示真实抓取、真实核验、真实生成的数据；链路失败或没有结果时，应展示空状态、候选状态或失败原因。
- 后续实现必须逐条对照本文确认过的样张和验收标准；若因技术、数据或成本限制无法达到，必须先说明差异并确认，不得悄悄打折扣。

## 六阶段总览

### 阶段 0：原型与内容合同

状态：当前阶段。

是否需要先和用户对原型：需要。

目标：先定义“用户应该看到什么”和“什么内容不能展示”，再动代码。

产物：

- 资料补全原型样张。
- 时效情报原型样张。
- 内容字段合同。
- 成卡/留候选/丢弃标准。
- 禁止展示清单。

### 阶段 1：资料补全闭环

是否需要先和用户对原型：需要，尤其是资料补全卡、空状态、这条不对弹窗和未成卡内容的后台处理方式。

范围：资料缺口、搜索意图、来源路线、正文清洗、后台候选事实、身份核验、缺口映射、AI 摘要、资料补全前端展示一起做。

验收目标：用户只看到已核验资料；如果没有已核验资料，页面说明当前还没有可用内容，不用候选线索填充主界面。

### 阶段 2：时效情报闭环

是否需要先和用户对原型：需要，尤其是情报类型、对象相关性、影响判断和建议动作。

范围：时效情报搜索意图、类型判别、对象相关性、时间窗口、影响分析、建议动作、任务/计划/事件线关联、时效情报前端展示一起做。

验收目标：每张时效情报卡都能回答“为什么我现在应该看这条”。

### 阶段 3：运行策略与用户校准

是否需要先和用户对原型：需要，尤其是关注指令、抓取频率、补充判断标准、关注后续的交互。

范围：重点关注指令、轻量/标准/高关注抓取模式、不采纳、补充判断标准、关注后续、追问、转任务与反馈学习一起做。

验收目标：用户每个动作都能改变后续判断方向或工作流，而不是只改变当前卡片状态。

### 阶段 4：默认来源包与质量评估

是否需要先和用户对原型：若只做后台策略，先对技术方案；若展示来源健康或覆盖状态，需要先对原型。

范围：不同资料缺口和情报类型的来源路线、官网发现、栏目下钻、多搜索引擎兜底、来源健康、失败重试、重复合并、内容质量评估一起做。

验收目标：来源扩展服务于“更可靠的工作判断”，而不是追求更多搜索结果。

### 阶段 5：规模化与回归测试

是否需要先和用户对原型：不需要视觉原型，需要确认验收清单。

范围：样张测试、反例测试、资料/时效混流测试、内部术语泄漏测试、样张不得入库测试、批量对象性能和抓取预算一起做。

验收目标：后续修改不能退回“搜索短摘包装成卡片”的状态。

## 阶段 0：内容原型 v0.2

以下样张只作为设计和验收基准，不得写入产品数据。

### 0.1 顶部与页签

顶部只保留用户需要主动使用的控制：

```text
客户/项目情报流

工作对象：广东省日慈公益基金会
时间排序：最新发现优先

[立即补全资料] [立即抓取情报]
```

不保留“刷新列表”。页面数据应随筛选、触发和后台状态自动刷新；若后续确需手动重读，只能作为低优先级辅助动作，不进入主原型。

系统状态不做单独横幅，改为跟随页签的关键词：

```text
[资料补全 12]
最近抓取：05/13 11:24 ｜ 累计已核验资料：12 条 ｜ 本次新增：2 条

[时效情报 3]
最近抓取：05/13 11:20 ｜ 累计情报：3 条 ｜ 本次新增：1 条
```

页签说明单独一行高亮显示：

```text
资料补全会把外部公开信息整理成可复用事实，用于补齐客户/项目画像、公开资料、项目事实和数据中心证据。这里展示的内容已经过系统核验；如果发现不准确或不需要，可以标记“这条不对”。
```

```text
时效情报用于提醒你近期可能值得判断、跟进或转给同事阅读的外部变化，例如资助窗口、政策变化、招标采购、合作方动态、行业风险和同类机构动作。
```

### 0.2 资料补全内容合同

普通用户主界面只展示系统核验过的信息，不展示候选线索区。

资料补全卡固定字段：

- 客户/项目。
- 补足资料。
- 可复用事实。
- 来源。
- 这条不对。

资料补全卡不展示：

- 摘要。
- 核验依据。
- 状态。
- 查看来源按钮。
- 候选线索普通展示区。
- 置信百分比、强来源、双证候选等内部判断。

来源文本本身必须可点击跳转网页；多个来源换行分开。

```text
- 可用于后续来源优先级：同一客户/项目的官网页面优先于普通搜索结果
```

点击“这条不对”后打开弹窗：


### 0.3 时效情报内容合同

时效情报卡固定字段：

- 情报类型。
- 关联对象。
- 时效性。
- 标题。
- 发生了什么。
- 为什么和你有关。
- 可能影响。
- 建议动作。
- 来源。
- 追问。
- 转任务。
- 关注后续。
- 不采纳。

不做单独收藏。重要但不执行的内容，也通过“转任务”变成阅读/研判任务，交给相关同事或自己后续处理。

#### 时效情报样张 A：资助机会

```text

标题

发生了什么

可能影响
- 机会：可作为儿童心理健康服务项目的资金或合作入口。
- 约束：需要确认是否接受基金会直接申报，还是仅接受执行机构申报。
- 风险：如果地域、主体资格或预算上限不匹配，不应投入过多准备成本。

建议动作
建议转成任务，交由相关同事核验申报资格、截止时间、地域限制、预算上限和是否允许联合申报。

来源
资助方官网公告
https://example.org/grant

[追问] [转任务] [关注后续] [不采纳]
```

#### 追问改法与验收标准

每张时效情报卡不展示固定推荐问题。点击“追问”后，弹窗可预置更深的问题方向，帮助用户围绕当前情报继续判断。

通用追问方向：

1. 这条外部变化会通过什么链条影响当前客户/项目？
2. 它和当前客户/项目的业务、对象、地域、资源需求分别有多强相关？
3. 如果要跟进，最关键的不确定点是什么？
4. 它更像机会、风险、约束，还是行业趋势？
5. 哪些证据不足，暂时不能据此行动？

针对行业风险，可追问：

- 这类监管提醒可能通过什么链条传导到当前客户/项目？
- 如果当前对象缺少相关资质，哪些项目传播表述最容易踩线？
- 这个风险是立即需要处理，还是只需要纳入材料审核清单？
- 需要检查哪些现有材料，才能判断它是否真的相关？

针对公益创投/资助机会，可追问：

- 资助方向和当前客户/项目关注对象的相关度有多高？
- 如果只看申报方向，当前对象更适合直接申报，还是作为资助方/合作方参与？
- 这条机会需要哪些已有资料支撑？哪些资料目前还缺？
- 申报窗口和当前项目准备度之间的缺口有多大？

针对合作方动态，可追问：

- 这条合作方动态说明资助方或合作方偏好发生了什么变化？
- 它对当前客户/项目的项目设计有什么启发？
- 它对益语智库的服务方案有什么可转化价值？
- 是否值得转成同事阅读任务，而不是立即执行任务？

追问验收标准：

- 必须围绕“传导链条、相关度、判断分歧、证据缺口、下一步决策”展开。
- 不得只提供“这是什么、有哪些要点、怎么跟进”这类浅层问题。
- 不同情报类型应生成不同的追问角度。
- 追问接口必须尽量把当前情报的类型、关联对象、资料标签、建议动作带入上下文，让回答围绕“这条变化到底怎么影响这个对象”，而不是泛泛解释网页。

时效情报点击“不采纳”后打开弹窗：

```text
为什么不采纳？

选项：
[无关] [不准确] [重复] [过期] [低价值]

补充说明：
[文本框]
例如：这只是泛行业新闻，和日慈当前项目无关；以后少推类似内容。

[取消] [提交]
```

提交后：

- 当前卡片隐藏。
- 记录到当前客户/项目的情报判断标准。
- 后续同类主题、来源或表达方式降权。

### 0.4 我重点关注什么

```text
我重点关注什么

生效范围：广东省日慈公益基金会

资料补全优先
- 机构登记和公开基础资料
- 年报、审计报告、年度预算
- 儿童心理健康项目案例
- 合作方和资助关系

时效情报优先
- 儿童心理健康资助窗口
- 广东省及珠三角公益合作机会
- 政府购买服务、民政相关政策
- 同类基金会动作

少看或不看
- 泛行业新闻
- 搜索引擎图片、地图、聚合页
- 和广东/儿童心理健康/基金会业务无关的政策通知

[保存关注指令]
```

关注指令应影响下一轮搜索意图、来源路线、核验规则和成卡标准，而不是只作为前端备注。

### 0.5 禁止展示清单

普通用户页面不应展示：

- 候选线索普通展示区。
- 搜索短摘原文堆叠。
- JS、CSS、导航、地图脚本、搜索引擎页噪音。
- 抓取报错码或英文异常作为主内容。
- 内部字段，如 `web_search`、`gov_policy`、`source_tier`、`confidence_score`。
- “强来源”“双证候选”“置信 76%”等未解释的内部判断。
- “已核验”“已进入数据中心”等在每张资料卡上反复出现的状态提示。
- “查看来源”按钮；来源文本本身应可点击。
- 模板化空话，例如“先确认负责人，再补关键材料，最后写回任务与手册”。
- 与当前对象无关的泛政策、泛通知、泛行业新闻。

### 0.6 成卡标准

资料补全成卡必须同时满足：

- 命中客户/项目身份锚点。
- 能映射到明确资料缺口或标签。
- 能提炼出详细、可复用事实。
- 来源可靠或有足够核验依据。
- 不依赖候选短摘、脏原文或搜索结果页。

时效情报成卡必须同时满足：

- 内容确实具有时效性或近期判断价值。
- 和当前对象有明确关系。
- 能说清影响、机会、风险、约束或可借鉴点。
- 能给出具体建议动作，包括执行任务或阅读/研判任务。

不满足成卡标准的内容，只能留在后台候选池、低优先级素材或直接丢弃，不进入普通用户主界面。
