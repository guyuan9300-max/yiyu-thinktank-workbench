# 组织搭建画布设计规格

## 1. 页面定位

这不是普通组织架构图，而是：

`组织搭建画布 / AI 可读组织建模台`

目标：
- 给管理者一个可查看、可编辑、可补全的组织工作台
- 给 AI 一套可读取的组织结构、职责、流程、项目关系底图

它要同时回答：
1. 当前组织由哪些部门组成
2. 每个部门下有哪些岗位、成员、计划
3. 哪些地方还没搭完
4. 哪些模板、流程、负责人还缺失
5. 项目和组织如何关联

## 2. 总体原则

1. 不只做树状图，要做可编辑画布
2. 不用树图替代现有卡片，而是做多视图切换
3. 树只处理：组织 -> 部门 -> 岗位 -> 成员
4. 项目属于横向关系，不硬挂在部门树下
5. 支持拖拽，但必须有层级吸附和自动排版
6. 风格延续当前产品：蓝白灰、轻量、圆角、留白
7. 页面既服务人，也服务 AI

## 3. 页面结构

分 3 区：

1. 顶部工具栏
2. 中间主画布
3. 右侧详情抽屉

### 3.1 顶部工具栏

左侧：
- 标题：`组织搭建画布`
- 副标题：`边设计组织，边补岗位，边接流程，边让 AI 读懂组织`

中间：
- 视图切换：
  - `卡片视图`
  - `树状图视图`
  - `协作关系视图`
- 搜索框：搜部门、岗位、成员、项目
- 筛选器：
  - `只看部门`
  - `只看岗位`
  - `只看成员`
  - `只看未完成节点`
  - `只看待补全模板`

右侧：
- `放大`
- `缩小`
- `适配画布`
- `自动排版`
- `新增` 下拉：
  - `新增部门`
  - `新增岗位`
  - `新增成员`
  - `新增项目关联`
- `保存`

## 4. 视图定义

### 4.1 卡片视图
- 保留当前“组织树与完整度”
- 适合看部门建设状态、负责人、完整度、缺口

### 4.2 树状图视图
- 第一阶段主视图
- 只展示：
  - 组织
  - 部门
  - 岗位
  - 成员
- 支持展开/折叠、拖拽、右侧抽屉编辑

### 4.3 协作关系视图
- 第一阶段先预留框架
- 第二阶段接入项目节点和虚线关系

## 5. 节点设计

### 5.1 组织节点
显示：
- 组织名称
- 部门总数
- 岗位总数
- 成员总数
- 整体完整度
- 主按钮：`新增部门`

### 5.2 部门节点
显示：
- 部门名称
- 部门负责人
- 完整度百分比
- 岗位数
- 成员数
- 计划数
- 状态标签：
  - `负责人已认领`
  - `已有岗位模板`
  - `待接入人岗计划`
  - `缺岗位流程`
  - `缺岗位说明`

支持：
- 展开岗位
- 折叠岗位
- 快速新增岗位
- 同级拖拽排序
- 点击打开抽屉

### 5.3 岗位节点
显示：
- 岗位名称
- 所属部门
- 当前成员数
- 当前负责人
- 岗位说明状态
- 岗位流程状态
- 模板状态
- 是否空缺

状态标签：
- `已有岗位说明`
- `已有岗位流程`
- `缺模板`
- `待招募`
- `已绑定成长路径`

支持：
- 展开成员
- 折叠成员
- 快速新增成员
- 可拖到其他部门下
- 点击打开抽屉

### 5.4 成员节点
显示：
- 姓名
- 当前岗位
- 当前项目数
- 当前任务负荷：`正常 / 偏满 / 空闲`
- 技能标签或成长状态

支持：
- 拖到其他岗位下
- 点击打开抽屉

## 6. 项目呈现方式

项目不作为树形子节点。

第一阶段：
- 在右侧抽屉显示 `关联项目`

第二阶段：
- 在 `协作关系视图` 中，把项目作为独立节点
- 用虚线连接到：
  - 部门
  - 岗位
  - 成员

原则：
- 树解决归属
- 项目关系解决协作

## 7. 右侧详情抽屉

### 部门抽屉
显示：
- 部门名称
- 部门职责描述
- 部门负责人
- 当前成员
- 当前岗位
- 计划中的岗位
- 完整度
- 缺失项提示
- 关联项目
- AI 建议（预留）

按钮：
- `编辑部门`
- `新增岗位`
- `绑定负责人`
- `补岗位模板`
- `补岗位流程`

### 岗位抽屉
显示：
- 岗位名称
- 所属部门
- 岗位职责
- 岗位说明书状态
- 岗位流程状态
- 负责人
- 成员列表
- 关联任务类型
- 关联成长路径
- 模板 / SOP / 学习资料

按钮：
- `编辑岗位`
- `新增成员`
- `绑定模板`
- `绑定流程`
- `绑定成长路径`

### 成员抽屉
显示：
- 姓名
- 所属岗位
- 所属部门
- 当前参与项目
- 当前任务量
- 当前成长状态
- 当前技能标签

按钮：
- `调整岗位`
- `查看任务`
- `查看成长手册`
- `解除归岗`

## 8. 拖拽规则

允许：
1. 同级部门之间调整顺序
2. 岗位拖到其他部门下
3. 成员拖到其他岗位下
4. 项目关联拖拽（预留）

不允许：
1. 成员直接拖成部门
2. 岗位挂到成员下
3. 节点自由漂浮失去层级

必须提供：
- 自动排版
- 层级吸附
- 拖拽预览高亮
- 拖拽后确认提示
- 一键恢复整齐布局

## 9. 完整度机制

必须保留并强化。

建议：
- 部门节点显示完整度进度徽标
- 岗位节点显示缺失项：
  - `缺岗位说明`
  - `缺流程模板`
  - `缺负责人`
  - `缺成员`
- 顶部显示全组织建设完成度
- 支持 `只看未完成节点`
- hover 显示完整度构成

## 10. 数据模型建议

在现有组织模型基础上新增：

1. `org_canvas_view_state`
- 当前视图
- 缩放比例
- 展开节点
- 当前筛选
- 当前选中节点

2. `org_canvas_layouts`
- 节点类型
- 节点 id
- 父节点
- 同级顺序
- 关系视图预留位置

3. `org_project_relations`
- 项目 id
- 目标类型：`department / role / member`
- 目标 id
- 关系类型：`owner / participant / reviewer / support`
- 权重

- `OrganizationCanvasPage`

抽屉级：

- `RoleDetailDrawer`

## 12. 接口建议

新增：
- `GET /api/v1/settings/org-model/canvas-data`
- `GET /api/v1/settings/org-model/canvas-state`
- `PATCH /api/v1/settings/org-model/canvas-state`
- `GET /api/v1/settings/org-model/canvas-layout`
- `PATCH /api/v1/settings/org-model/canvas-layout`
- `POST /api/v1/settings/org-model/canvas-layout/auto-arrange`
- `GET /api/v1/settings/org-model/project-relations`
- `POST /api/v1/settings/org-model/project-relations`
- `PATCH /api/v1/settings/org-model/project-relations/{relationId}`
- `DELETE /api/v1/settings/org-model/project-relations/{relationId}`

## 13. 与现有系统的嫁接

这页必须直接接：
1. 任务系统：复用 `task_org_links`
2. 周计划：部门计划挂到部门节点
3. 周总结：部门和 CEO 总结读取完整度与组织结构
4. 成长系统：岗位挂成长路径、模板、学习资料
5. 机器人员工：机器人作为正式成员节点进入结构图

## 14. 开发分期

### P1：可用版
- 三种视图切换
- 树状图主视图
- 节点展开/折叠
- 右侧详情抽屉
- 部门/岗位/成员拖拽
- 完整度可视化

### P2：增强版
- 自动排版
- 缩放 / 适配画布
- 搜索和筛选
- 画布状态持久化
- 抽屉里的关联项目

### P3：关系图版
- 协作关系视图
- 项目独立节点
- 虚线关系
- AI 建议面板
- 完整度 hover 细分

## 15. 成功标准

用户打开这页时，应明显感到：

`我不是在看一张冷冰冰的组织架构图，而是在搭一套活的组织系统。部门、岗位、成员、流程、模板、项目关系都能在这里长出来，而且这套结构还能被 AI 读懂。`
