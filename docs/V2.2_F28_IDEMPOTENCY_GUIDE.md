## 0 · 一句话目标

**让任何写入操作(POST/PATCH)在客户端 retry 时不重复创建数据。** 同一个 `Idempotency-Key` 第二次过来,直接返回第一次的结果。

这是 3.0 AI agent 必备的容错基础 — AI 自动 retry 失败请求时,**不能**导致数据重复。

## 2 · 使用模式 (Stripe 风格)

### 2.1 endpoint 视角

```python
@app.post("/api/v1/tasks", response_model=TaskRecord)
def create_task(
    payload: TaskCreatePayload,
    idempotency_key: str | None = Header(None, alias="Idempotency-Key"),
    actor_type: str = Header("human", alias="X-Actor-Type"),
    actor_id: str = Header("", alias="X-Actor-Id"),
) -> TaskRecord:
    from app.services.idempotency_store import (
        IdempotencyKeyMismatchError,
        get_idempotency_store,
    )

    # 没带 key → 不走幂等,直接处理(向后兼容)
    if not idempotency_key:
        return _create_task_business_logic(payload)

    store = get_idempotency_store(state.db)
    METHOD, PATH = "POST", "/api/v1/tasks"

    # Step 1: 检查缓存
    try:
        cached = store.find(idempotency_key, METHOD, PATH, payload=payload.dict())
    except IdempotencyKeyMismatchError as exc:
        raise HTTPException(status_code=422, detail=str(exc))

    if cached:
        if cached.status == "completed":
            # 直接返回缓存,不重新创建
            return TaskRecord.parse_raw(cached.response_body)
        elif cached.status == "in_progress":
            # 上次请求还在处理(并发 retry)
            raise HTTPException(
                status_code=409,
                detail="Request still in progress, retry later",
            )
        # status == 'failed' → 当作新请求 (落到下面 start)

    # Step 2: 标记开始处理
    store.start(
        idempotency_key, METHOD, PATH,
        payload=payload.dict(),
        actor_type=actor_type,
        actor_id=actor_id,
    )

    # Step 3: 正常业务
    try:
        result = _create_task_business_logic(payload)
    except Exception:
        store.mark_failed(idempotency_key, METHOD, PATH)
        raise

    # Step 4: 缓存响应
    store.complete(
        idempotency_key, METHOD, PATH,
        status=201,
        response_body=result.dict(),
    )
    return result
```

### 2.2 客户端视角

```typescript
// 任何 POST/PATCH 加 Idempotency-Key header
async function createTaskSafely(payload: TaskCreatePayload) {
  const key = crypto.randomUUID();  // 客户端生成
  let attempt = 0;
  while (attempt < 3) {
    try {
      return await request<TaskRecord>("/api/v1/tasks", {
        method: "POST",
        headers: { "Idempotency-Key": key },
        body: JSON.stringify(payload),
      });
    } catch (err) {
      if (err.status === 409) {
        // 上次请求还在处理,稍等再 retry (同一个 key)
        await sleep(500);
        attempt++;
        continue;
      }
      throw err;
    }
  }
}
```

## 3 · AI Agent 场景 (N3 核心)

3.0 AI agent 自动 retry 时,header 应该这样:

```
POST /api/v1/tasks
Idempotency-Key: ai-agent-contract-drafter-2026-05-22-task-001
X-Actor-Type: ai_agent
X-Actor-Id: agent_contract_drafter
Content-Type: application/json

{ "title": "AI 起草: 跟进供应商合同", "owner_id": "user_guyuan" }
```

`actor_type='ai_agent'` 让 idempotency_keys 表记录是 AI 自动写入还是人类操作 — 这是 3.0 审计的关键字段。

## 4 · 渐进接入路线 (下个 session)

不需要一次性给所有 endpoint 加。**从 P0 风险高的端点开始**:

| 优先级 | endpoint | 理由 |
|---|---|---|
| **P0** | `POST /api/v1/tasks` | 主任务创建,客户端最常 retry |
| **P0** | `POST /api/v1/event-lines` | 事件线创建 |
| **P0** | `POST /api/v1/clients` | 客户创建 |
| **P1** | `POST /api/v1/tasks/{id}/attachments` | 附件上传,网络断了用户必然 retry |
| **P1** | `PATCH /api/v1/tasks/{id}` | 更新场景多 |
| **P2** | `POST /api/v1/consultation/chat` | 长流程,中间断了不希望重新跑 LLM |
| **不做** | `GET *` | 幂等天然 GET 不需要 |

**接入工作量**: 每个 endpoint ~10 行代码模式化套入,但需要从 payload 里识别哪些字段是 idempotency-relevant(比如 task.title + owner_id 是,内部 _generated_at 不是)。

每接 1 个端点跑一次:
```bash
cd backend && .venv/bin/python3 -m pytest ../tests/test_idempotency_store.py -q
```

确保没破坏 store 行为。

## 5 · cleanup 后台任务 (建议但不强制)

idempotency_keys 24 小时后过期。需要定期跑 `store.cleanup_expired()` 清理表,否则会无限增长。

建议在 v2.2 backend startup 时挂一个后台 cron(每小时跑一次):

```python
async def _idempotency_cleanup_loop():
    while True:
        try:
            n = get_idempotency_store(state.db).cleanup_expired()
            if n > 0:
                logger.info("idempotency cleanup deleted %d rows", n)
        except Exception:
            logger.exception("idempotency cleanup failed")
        await asyncio.sleep(3600)
```
