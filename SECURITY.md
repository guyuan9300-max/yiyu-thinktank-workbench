# Security Policy

## 支持范围

当前开源仓库只接受最新 `main` 分支和最新公开 release 的安全反馈。

## 私下报告安全问题

如果你发现以下问题，请不要直接公开 issue：

- 密钥、token、证书或数据库泄露。
- 越权访问、跨组织数据可见、权限绕过。
- 远程代码执行、命令注入、任意文件读写。
- 飞书、对象存储、AI 或云端集成的敏感信息泄露。

请通过以下方式私下联系维护者：

- GitHub Security Advisories（公开后启用）
- 邮箱：security@example.com

邮件中请包含复现步骤、影响范围、版本号和建议修复方向。请不要附带真实用户数据或未脱敏客户资料。

## 密钥处理原则

- 真实 `.env`、云端密钥、飞书密钥、对象存储 AK/SK、Apple 证书和公证配置不得提交到仓库。
- 如果密钥曾进入 git 历史，必须先轮换密钥，再清理历史记录，最后重新扫描。
- 示例配置只能使用占位符。
